Dimensionality Reduction
Compress high-dimensional data into fewer dimensions — removing noise, revealing structure, enabling visualization, and speeding up downstream models.
PCA — Principal Component Analysis
Find orthogonal directions of maximum variance in the data via eigendecomposition of the covariance matrix.
$$\begin{aligned} \Sigma &= \tfrac{1}{n}X^TX \quad \text{(covariance matrix, after mean-centering)} \\ \Sigma \mathbf{v} &= \lambda \mathbf{v} \quad \text{(eigendecomposition)} \\ \mathbf{PC}_1 &= \text{eigenvector with largest eigenvalue} \\ \mathbf{PC}_2 &= \text{eigenvector with 2nd largest (orthogonal to } \mathbf{PC}_1) \\ \text{Explained variance ratio} &= \lambda_i \;/\; \textstyle\sum_j \lambda_j \end{aligned}$$
  • How many components? Choose k such that cumulative explained variance ≥ 95%
  • Assumes linear relationships between features
  • Interpretability suffers: PCs are linear combinations of all features
  • Standardize features first — PCA is scale-sensitive
t-SNE
t-Distributed Stochastic Neighbor Embedding. Non-linear, preserves local structure, excellent for 2D/3D visualization.
  • Converts distances to probabilities; minimizes KL divergence between high-D and low-D probability distributions
  • Perplexity parameter (5–50): controls neighborhood size
  • Not for features: t-SNE is only for visualization — distances in 2D plot don't have meaningful scale
  • Non-deterministic; computationally expensive O(n²)
UMAP
Uniform Manifold Approximation and Projection. Faster than t-SNE, preserves global structure better.
  • Based on Riemannian geometry and fuzzy topology — mathematically rigorous
  • Preserves both local AND global structure (t-SNE loses global)
  • Much faster: O(n log n) vs O(n²)
  • Can be used for feature engineering (unlike t-SNE)
  • Parameters: n_neighbors (local), min_dist (spread)

🎯 Interview Gold

"I use PCA as a preprocessing step when features are highly correlated (reduces multicollinearity for linear models) or when I need to reduce compute. For visualization, UMAP is now my default over t-SNE — it's faster, preserves global structure, and can handle larger datasets. A critical gotcha: always fit PCA on training data only, then transform both train and test — otherwise you leak test information into the representation."
