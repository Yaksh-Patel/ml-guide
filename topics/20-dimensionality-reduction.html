Dimensionality Reduction
Compress high-dimensional data into fewer dimensions — removing noise, revealing structure, enabling visualization, and speeding up downstream models.
PCA — Principal Component Analysis
Find orthogonal directions of maximum variance in the data via eigendecomposition of the covariance matrix.
$$\begin{aligned} \Sigma &= \tfrac{1}{n}X^TX \quad \text{(covariance matrix, after mean-centering)} \\ \Sigma \mathbf{v} &= \lambda \mathbf{v} \quad \text{(eigendecomposition)} \\ \mathbf{PC}_1 &= \text{eigenvector with largest eigenvalue} \\ \mathbf{PC}_2 &= \text{eigenvector with 2nd largest (orthogonal to } \mathbf{PC}_1) \\ \text{Explained variance ratio} &= \lambda_i \;/\; \textstyle\sum_j \lambda_j \end{aligned}$$
  • How many components? Choose k such that cumulative explained variance ≥ 95%
  • Assumes linear relationships between features
  • Interpretability suffers: PCs are linear combinations of all features
  • Standardize features first — PCA is scale-sensitive
t-SNE
t-Distributed Stochastic Neighbor Embedding. Non-linear, preserves local structure, excellent for 2D/3D visualization.
  • Converts distances to probabilities; minimizes KL divergence between high-D and low-D probability distributions
  • Perplexity parameter (5–50): controls neighborhood size
  • Not for features: t-SNE is only for visualization — distances in 2D plot don't have meaningful scale
  • Non-deterministic; computationally expensive O(n²)
UMAP
Uniform Manifold Approximation and Projection. Faster than t-SNE, preserves global structure better.
  • Based on Riemannian geometry and fuzzy topology — mathematically rigorous
  • Preserves both local AND global structure (t-SNE loses global)
  • Much faster: O(n log n) vs O(n²)
  • Can be used for feature engineering (unlike t-SNE)
  • Parameters: n_neighbors (local), min_dist (spread)

Common Mistakes & Misconceptions
  • Fitting PCA on the full dataset before train/test split — PCA fitted on test data leaks test distribution into the components. Always fit PCA on training data only, then transform both train and test with the same components
  • Using t-SNE for anything other than visualization — t-SNE distances in 2D have no meaning: distances between clusters are arbitrary, angles are meaningless, and the same data with different perplexity produces very different layouts. Never use t-SNE embeddings as features for downstream models
  • Choosing n_components arbitrarily — for PCA, plot explained variance ratio vs component count; choose the "elbow" or the point where cumulative variance ≥ 95%. Never pick a round number (e.g., 10) without justification
  • Applying PCA to categorical/binary features — PCA is designed for continuous features and assumes linear relationships. Use MCA (Multiple Correspondence Analysis) for categorical data, or encode then normalise before PCA
  • Assuming dimensionality reduction always improves model performance — PCA discards variance, which can include signal. Always compare model performance with and without PCA on a validation set; it's a preprocessing choice, not a guaranteed improvement
How It's Used at Top Companies
  • Spotify (music embeddings): PCA on audio feature vectors (tempo, energy, valence, danceability) to compress 20+ features into 5–6 principal components for clustering and recommendation retrieval
  • Netflix (content understanding): Truncated SVD (LSA) on movie-tag co-occurrence matrices; PCA on user-rating matrices for collaborative filtering — standard dimensionality reduction in classic recommender systems
  • 10x Genomics / biotech: PCA then UMAP on single-cell RNA-seq data (20,000+ gene dimensions → 2D); standard pipeline for visualizing cell type clusters in computational biology
  • Finance (risk management): PCA on correlated asset return matrices to extract independent risk factors; the first few principal components often correspond to market, sector, and style factors
🎯 Interview Gold

What to say: "I use PCA when features are highly correlated — it removes multicollinearity and reduces compute for downstream linear models. I fit PCA on training data only, choose n_components where cumulative explained variance ≥ 95%, and whittle down from there if compute is constrained. For visualization of high-dimensional embeddings, UMAP is now my default over t-SNE: faster, preserves global structure, and supports out-of-sample projection. Key gotcha: t-SNE output is not a feature space — distances are meaningless after projection."

Common mistake to avoid saying: "I applied PCA to get 50 components." — Without justifying component count via explained variance, this signals cargo-cult ML. Always justify n_components with a scree plot or explicit variance threshold.

Expect the follow-up: "What does each principal component represent?" — Answer: each PC is a linear combination of original features (a direction in feature space) chosen to maximise variance in that direction, subject to orthogonality to previous PCs. PC1 = direction of greatest variance, PC2 = next greatest orthogonal direction, etc. The loadings (feature weights in each PC) show which original features contribute most to each component.
