Loss Functions
The objective you optimize — every loss function encodes a probability model and a cost structure. Choose wrong and you optimize for the wrong thing. For classifiers, cross-entropy is almost always the right choice over MSE or raw classification error.
Regression Losses
$$\begin{aligned} \text{MSE} &= \tfrac{1}{n}\sum(y-\hat{y})^2 \quad\rightarrow\text{assumes Gaussian noise} \\ \text{MAE} &= \tfrac{1}{n}\sum|y-\hat{y}| \quad\rightarrow\text{assumes Laplace noise} \\ \text{Huber} &= \begin{cases} \tfrac{1}{2}(y-\hat{y})^2 & \text{if } |e| \leq \delta \\ \delta|y-\hat{y}|-\tfrac{\delta^2}{2} & \text{otherwise}\end{cases} \end{aligned}$$
  • MSE: penalizes large errors heavily; differentiable; outlier-sensitive
  • MAE: robust to outliers; non-differentiable at 0 (subgradient)
  • Huber: MSE near 0, MAE for large errors — best of both
  • Log-cosh: smooth approximation to Huber; twice differentiable everywhere
Classification Losses
$$\begin{aligned} BCE = -\left[y \log \hat{y} + (1-y) \log(1-\hat{y})\right] \\ CCE = -\sum y_i \log \hat{y}_i \quad \text{[categorical cross-entropy]} \\ ACE = -(1/n) \sum \log(\hat{y}_{\text{correct}}) \quad \text{[average cross-entropy]} \\ Hinge = \max(0, 1-y\cdot\hat{y})_{SVMloss} \\ Focal = -(1-\hat{y})^\gamma \log \hat{y} \quad \text{[imbalanced data]} \end{aligned}$$
  • BCE: standard binary; equivalent to Bernoulli MLE
  • CCE / ACE: multi-class with softmax; equivalent to Categorical MLE
  • Focal Loss: (1−ŷ)^γ down-weights easy examples; γ=2 is standard
  • Label smoothing: target = 0.9 instead of 1.0 — prevents overconfident predictions
Why Average Cross-Entropy (ACE) Beats Classification Error and MSE
Consider two classifiers predicting political affiliation (Democrat / Republican / Other) — both get 2 out of 3 examples correct, so both have identical classification error = 1/3. But they are clearly not equally good:
Classifier A — barely correct
Computed | Target | Correct? 0.3 0.3 0.4 | 0 0 1 ✓ | yes (barely) 0.3 0.4 0.3 | 0 1 0 ✓ | yes (barely) 0.1 0.2 0.7 | 1 0 0 ✗ | no (way off)
Classifier B — confidently correct
Computed | Target | Correct? 0.1 0.2 0.7 | 0 0 1 ✓ | yes (confident) 0.1 0.7 0.2 | 0 1 0 ✓ | yes (confident) 0.3 0.4 0.3 | 1 0 0 ✗ | no (barely off)
Classification error is blind to how confident the prediction was. ACE is not — it uses ln() to reward confident correct predictions and heavily penalize wrong ones.
$$ACE = -\dfrac{1}{n}\sum_i \ln\bigl(\hat{y}_{\text{correct},i}\bigr)$$
For each example, only the log-prob of the TRUE class is counted
(other terms multiply by 0 from one-hot target and vanish)

Classifier A: −(ln(0.4)+ln(0.4)+ln(0.1))/3
   = −(−0.916 + −0.916 + −2.303)/3 = 1.38 ← higher = worse

Classifier B: −(ln(0.7)+ln(0.7)+ln(0.3))/3
   = −(−0.357 + −0.357 + −1.204)/3 = 0.64 ← lower = better ✓
ACE correctly identifies Classifier B as superior. Classification error cannot.
MSE for Classifiers — Better Than Accuracy, Still Worse Than ACE
$$\text{MSE} = \dfrac{1}{n}\sum_i \sum_j (y_{ij} - \hat{y}_{ij})^2 \quad \text{(summed over all output neurons)}$$
Clf A, ex1: (0.3−0)²+(0.3−0)²+(0.4−1)² = 0.09+0.09+0.36 = 0.54
Clf A, ex3: (0.1−1)²+(0.2−0)²+(0.7−0)² = 0.81+0.04+0.49 = 1.34
MSE Clf A = (0.54+0.54+1.34)/3 = 0.81

Clf B, ex1: (0.1−0)²+(0.2−0)²+(0.7−1)² = 0.01+0.04+0.09 = 0.14
Clf B, ex3: (0.3−1)²+(0.4−0)²+(0.3−0)² = 0.49+0.16+0.09 = 0.74
MSE Clf B = (0.14+0.14+0.74)/3 = 0.34
MSE also correctly ranks B above A, but it wastes signal on the wrong neurons — it penalizes the 0.3 and 0.4 outputs of the incorrect class, even though those are irrelevant to the decision. ACE only grades you on the probability you assigned to the true class, which is exactly the information that matters.
The Deeper Reason: MSE Causes Training to Stall Near Convergence
$$\Delta W \propto (\hat{y}-y)\cdot\underbrace{\hat{y}(1-\hat{y})}_{\sigma'(z)\text{ term}} \quad \text{(sigmoid + MSE)}$$
When ŷ̂ → 1.0 (confident & correct):  0.95·(1−0.95) = 0.0475
When ŷ̂ → 0.5 (uncertain):             0.5·(1−0.5)  = 0.25
As network improves, σ′(z) → 0 → gradient → 0 → weights barely move.
Training stalls even though not fully converged.
$$\Delta W \propto (\hat{y}-y) \quad \text{(cross-entropy + softmax/sigmoid \u2014 } \sigma'\text{ cancels!)}$$
Gradient ∝ prediction error only. No stalling. The cancellation is intentional.
\end{aligned}$$
  • This cancellation is why BCE is the canonical loss for binary classification with sigmoid output
  • The same cancellation occurs for CCE with softmax — ∂L/∂z = ŷ − y
  • This is not a coincidence — these loss-activation pairs are designed together to have clean gradients

📖 Taxi Meter Analogy

MSE is a taxi that charges exponentially per mile you're off route: 1 mile wrong = $1, 10 miles = $100. MAE charges linearly. For a city with occasional crazy detours (outliers), MAE is fairer. Huber is a taxi with a daily cap. Cross-entropy is a different kind of taxi — it only charges you for how confident you were about the street you recommended. If you confidently said "go left" and the destination was right, the fare is astronomical. If you quietly suggested "maybe left?" it's a smaller penalty. It rewards learning to be confident about the right things.

⚠️ Common Mistake

Using MSE as a classification loss (e.g., targets as 0/1 and computing squared error across all output nodes) is a subtly bad idea that used to be common. It works — MSE does rank classifiers in the right order — but it dilutes the gradient with signal from the wrong output neurons, and it causes the vanishing-gradient stall near convergence. Always use cross-entropy for classifier training.

🎯 Interview Gold

Three loss metrics exist for classifiers: classification error, MSE, and average cross-entropy (ACE). Classification error is too crude — two networks with the same accuracy can be wildly different in quality because it ignores how confident each prediction was. MSE is better but still flawed — it wastes gradient on incorrect output neurons, and critically, its gradient contains a σ'(z) = ŷ(1−ŷ) term that shrinks toward zero as predictions get confident, causing training to stall. ACE is best — it only grades you on the probability assigned to the true class, and when paired with softmax/sigmoid, the derivative of the activation function cancels out of the gradient completely, leaving ∂L/∂z = ŷ − y. No stalling, no wasted signal. The probability interpretation is also clean: minimizing ACE is maximum likelihood estimation under a Categorical distribution. After training, you can switch back to reporting classification accuracy — that's ultimately what you care about for stakeholders.

Common mistake to avoid saying: "I used MSE loss for my classifier." — Cross-entropy is strictly better for classification: it directly optimises the probability assigned to the true class, avoids the vanishing-gradient stall, and has a cleaner probabilistic interpretation.

Expect the follow-up: "When would you use Huber loss instead of MSE?" — Answer: when regression targets have outliers. MSE penalises outliers quadratically — one bad data point can dominate the gradient. Huber behaves like MSE for small errors (|y−ŷ| < δ) and like MAE for large errors, bounding the maximum gradient contribution from any single example. δ is a hyperparameter tuned on the validation set.

Common Mistakes & Misconceptions
  • Using MSE for classification — MSE wastes gradient on incorrect output neurons and stalls as predictions get confident (σ'(z)→0). Always use cross-entropy for classification tasks
  • Confusing loss function and evaluation metric — you train with cross-entropy but evaluate with accuracy/F1/AUC-ROC. These are different things; the training loss need not be the business metric. Never train directly on accuracy (non-differentiable)
  • Using MSE for skewed regression targets — if revenue predictions have a handful of extreme values, those dominate MSE and warp the model. Log-transform targets or use Huber/MAE for robustness
  • Not normalizing multi-task losses — in multi-task learning, two tasks with different loss scales cause one task to dominate training. Always normalise individual losses (divide by their scale) and weight them explicitly before summing
  • Forgetting numerical stability of log(0) — log(0) = -∞ causes NaN in cross-entropy. Always clip predictions: log(max(ŷ, 1e-7)) in custom loss implementations
How It's Used at Top Companies
  • Google (object detection — Focal Loss): introduced by Facebook AI (RetinaNet), Focal Loss down-weights easy negatives: FL = −(1−ŷ)^γ · log(ŷ). At γ=2, a well-classified example (ŷ=0.9) contributes 100× less than a misclassified one — forces the model to focus on hard examples in extreme class imbalance
  • OpenAI (RLHF — reward model loss): Bradley-Terry pairwise ranking loss used to train reward models from human preference pairs: L = −log σ(r(preferred) − r(rejected)); the reward model learned from this loss then drives PPO training
  • Spotify / Netflix (recommendation — contrastive loss): InfoNCE / NT-Xent loss trains embedding models to pull similar items together and push dissimilar ones apart — the basis of SimCLR, CLIP, and modern two-tower retrieval models
  • Finance (quantile regression loss): pinball loss enables prediction intervals; predicting the 10th and 90th percentiles of demand gives inventory planners uncertainty bounds rather than just a point estimate
