Transformers
The architecture that ate the world — "Attention is All You Need" (2017) replaced RNNs across NLP, then spread to vision (ViT), protein folding (AlphaFold), audio, video, and more.
Encoder-Decoder Architecture
Transformer Encoder-Decoder — encoder processes input, decoder generates output
Positional Encoding
Self-attention is permutation-invariant — shuffling tokens gives the same output. Positional encodings inject position information into the embeddings.
$$\begin{aligned} PE(\text{pos}, 2i) &= \sin\!\left(\frac{\text{pos}}{10000^{2i/d_{\text{model}}}}\right) \\ PE(\text{pos}, 2i+1) &= \cos\!\left(\frac{\text{pos}}{10000^{2i/d_{\text{model}}}}\right) \\ \text{Input to each layer} &= \text{token embedding} + \text{positional embedding} \end{aligned}$$
Original Transformer used sinusoidal encodings. Modern LLMs use Rotary Position Embedding (RoPE) or ALiBi — these generalize better to sequences longer than training length.

Common Mistakes & Misconceptions
  • Confusing encoder-only, decoder-only, and encoder-decoder — encoder-only (BERT): bidirectional, good for classification/NER/embeddings, can't generate. Decoder-only (GPT): causal, generative, the dominant paradigm for LLMs. Encoder-decoder (T5, BART): best for seq2seq tasks (translation, summarization) where input and output are both structured sequences
  • Using BatchNorm in transformers — BatchNorm computes statistics over the batch dimension; for variable-length sequences this is unstable. Transformers use Layer Norm (statistics over the feature dimension within each token) — batch-size independent and stable with variable-length inputs
  • "Pre-LN vs Post-LN doesn't matter" — it does. Original Transformer: Post-LN (norm after residual add) — training unstable without warmup. Modern transformers (GPT, LLaMA): Pre-LN (norm before sublayer) — more stable training, no warmup required, enables training without careful LR tuning
  • Treating the KV cache as automatic — KV cache must be explicitly managed at inference time. Many serving bugs come from incorrect cache invalidation (stale cache from previous request) or running out of KV cache memory at long contexts. Monitor KV cache utilization in production
  • Ignoring positional encoding differences — learned absolute PE (BERT/GPT-2): fixed context length, doesn't generalise beyond training length. RoPE (LLaMA, GPT-NeoX): rotary, relative, extrapolates better. ALiBi: linear bias on attention logits, zero parameters, excellent length generalisation. Choice matters for long-context tasks
How It's Used at Top Companies
  • Google (BERT → production NLP): BERT fine-tuned for query understanding in Google Search — the Transformer's ability to model bidirectional context transformed relevance ranking quality in 2019; still a component in the search stack
  • OpenAI (GPT-4): decoder-only transformer at scale; sparse mixture-of-experts (MoE) architecture — only a subset of FFN experts activated per token, enabling 8× parameter count at ~2× FLOPs; production serving via continuous batching
  • Meta (LLaMA 3): decoder-only with RoPE positional encoding, GQA, SwiGLU activation, Pre-LN — these four architectural choices are now the standard recipe for open-source LLMs; trains on 15T tokens
  • Microsoft (GitHub Copilot): fine-tuned GPT decoder for code completion; prefix-suffix-middle (PSM) training objective — model trained to fill in the middle of code given both prefix and suffix context
🎯 Interview Gold

What to say: "The transformer's key insight: replace sequential recurrence with parallel attention over all positions — O(n²) computation but fully parallelisable on GPUs. The architectural choices: residual connections for gradient flow through depth; Layer Norm (not BatchNorm) for sequence-length-independent stability; Pre-LN (before sublayer) in modern models for stable training without warmup. The paradigm shift: pretrain on massive unsupervised data via next-token prediction → fine-tune on task-specific data. This self-supervised pretraining is why transformers dominate — they leverage internet-scale data with no labels."

Common mistake to avoid saying: "I'll use BERT for text generation." — BERT is encoder-only with bidirectional attention; it cannot autoregressively generate text. BERT: classification, NER, embeddings. GPT-style decoders: generation. T5/BART: seq2seq tasks with a structured output sequence.

Expect the follow-up: "Why do modern LLMs use Pre-LN instead of the original Post-LN?" — Answer: Post-LN (original Transformer) applies LayerNorm after the residual addition: output = LayerNorm(x + F(x)). This means the residual path accumulates unnormalised values — gradients through early layers can explode or vanish, requiring careful warmup. Pre-LN: output = x + F(LayerNorm(x)) — the normalised input to each sublayer keeps gradients stable regardless of depth, enabling training without warmup and at higher learning rates.
