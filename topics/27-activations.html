Activation Functions
The non-linearity that makes deep networks more than stacked linear algebra. Without activations, depth is useless. With the right activation, gradients flow freely.
Why They're Critical
W₃(W₂(W₁x)) = (W₃W₂W₁)x — stack of linear layers = one linear layer. Activation functions break this and allow networks to learn arbitrary functions. Choice also controls gradient flow (vanishing/exploding) during backpropagation.

Dying ReLU Problem & Fixes
If a neuron receives consistently negative input, its output is always 0 and its gradient is always 0 — it can never recover. "Dead neurons" contribute nothing to the network.
  • Leaky ReLU: α=0.01 for negative inputs — always has a non-zero gradient
  • ELU: smooth negative saturation — better gradient properties
  • Careful initialization + BatchNorm: prevents weights from going too negative initially
  • Lower learning rate: reduces risk of neurons being pushed into dead zone early
Common Mistakes & Misconceptions
  • Using sigmoid/tanh in hidden layers of deep networks — both saturate (gradient ≈ 0 at extremes), causing vanishing gradients in deep networks. Hidden layers: ReLU or GELU. Sigmoid belongs only at the output for binary classification; tanh can work in LSTMs (bounded activations help gating)
  • "ReLU is always safe" — ReLU can cause dead neurons if learning rate is too high or weights are initialised poorly. A neuron that receives only negative pre-activations outputs 0 forever. Use He initialisation + BatchNorm to mitigate; use Leaky ReLU if dead neurons are observed (monitor fraction of zero-activation units)
  • Using softmax where sigmoid is needed — softmax forces outputs to sum to 1 (mutual exclusion); correct for single-label multi-class. For multi-label classification (an image can have both "cat" AND "outdoors"), use independent sigmoid per class — softmax would incorrectly distribute probability mass
  • Applying ReLU after the final layer — ReLU clips negatives to zero; for regression output you need the full real line (linear activation). For probability output you need [0,1] (sigmoid) or sum-to-1 (softmax). Never apply ReLU to the output layer
  • Ignoring GELU for transformers — GELU (used in GPT, BERT) is smooth and probabilistically motivated: x·Φ(x) weights the input by the probability it is above 0 under a standard normal. Empirically outperforms ReLU on transformers; don't default to ReLU for attention-based architectures
How It's Used at Top Companies
  • OpenAI / Anthropic (LLMs): SwiGLU activation (a gated variant of GLU) used in LLaMA, PaLM, and Claude — replaces FFN's ReLU/GELU with a learned gating mechanism: SwiGLU(x,W,V) = Swish(xW) ⊗ (xV); consistently outperforms GELU in large-scale language model training
  • Google (BERT, T5): GELU throughout all hidden layers; empirically determined to outperform ReLU on NLP tasks in the original BERT ablation study — now the standard for transformer encoder architectures
  • ResNet / EfficientNet (computer vision): ReLU remains dominant in CNN architectures; Swish/SiLU used in EfficientNet and MobileNetV3 — smooth activations improve gradient flow in very deep convolutional networks
  • LSTMs / GRUs (sequence models): tanh for cell state (bounded: prevents exploding hidden states), sigmoid for gates (naturally outputs [0,1] gating values) — this specific combination is deliberate and important for LSTM stability
🎯 Interview Gold

What to say: "ReLU is my default for hidden layers — gradient=1 for positive inputs prevents vanishing gradients, and it's computationally cheap. For transformer architectures I use GELU (smoother gradient flow, empirically better in BERT/GPT ablations). For very large LLMs, SwiGLU has become standard (LLaMA, PaLM). Output activations are task-determined: sigmoid for binary classification, softmax for single-label multi-class, independent sigmoids for multi-label, linear for regression. Never ReLU at the output."

Common mistake to avoid saying: "I used softmax for multi-label classification." — Softmax forces outputs to sum to 1, implying mutual exclusion. Multi-label needs independent sigmoid per class — each output is an independent binary decision.

Expect the follow-up: "What is the dying ReLU problem and how do you fix it?" — Answer: a ReLU neuron dies when its pre-activation is always negative, making output=0 and gradient=0 — the neuron can never recover. Causes: high learning rate pushing weights negative, bad initialisation. Fixes: He initialisation (prevents initial negative activations), BatchNorm (keeps pre-activations centred), Leaky ReLU (α=0.01 preserves a small gradient even for negative inputs), or monitoring and resetting dead neurons.
