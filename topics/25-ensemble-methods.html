Ensemble Methods
Combine multiple models to outperform any individual — the technique behind every top Kaggle solution and many production systems.
Bagging (Bootstrap Aggregating)
Train same algorithm on different bootstrap samples. Average predictions. Reduces variance.
  • Models trained independently and in parallel
  • Works best when base model has high variance (overfitting)
  • Examples: Random Forest, Extra Trees
  • OOB evaluation: each training sample missed by ~37% of trees → free validation
Boosting
Train sequentially — each model focuses on previous model's errors. Reduces bias.
  • Models trained sequentially, each correcting its predecessor
  • Works best when base model has high bias (underfitting)
  • Examples: XGBoost, LightGBM, AdaBoost, CatBoost
  • Risk: sensitive to noisy labels (each boosted model doubles down on hard examples)
Stacking (Stacked Generalization)
Train multiple diverse base models. Use their predictions as features for a meta-model (blender) trained on held-out predictions.
$$\begin{aligned} &\text{Stage 1: train base models } \{M_1, M_2, \ldots, M_k\} \text{ on training data} \\ &\text{Stage 2 (out-of-fold): get predictions from each } M_i \text{ on validation folds} \\ &\text{Stage 3: train meta-model on } [M_1(x_i), M_2(x_i), \ldots, M_k(x_i)] \rightarrow y \\ &\text{Inference: stack base model predictions} \rightarrow \text{feed to meta-model} \end{aligned}$$
  • Meta-model learns which base model to trust on which inputs
  • Diversity is key: combine logistic regression + XGBoost + neural network
  • Risk of leakage: always use out-of-fold predictions, not in-sample predictions
Voting & Blending
Voting Classifier
  • Hard voting: majority vote on class labels
  • Soft voting: average predicted probabilities — almost always better
  • Weights can be assigned to better-performing models
Blending vs Stacking
  • Blending: simpler — hold out a fixed 20% for meta-model training
  • Stacking: use k-fold CV out-of-fold predictions — more data for meta-model, less leakage risk
  • Blending faster to implement; stacking more robust
Common Mistakes & Misconceptions
  • Training the meta-model on in-sample predictions — the #1 stacking mistake. Base models fitted on the full training set and used to generate training features for the meta-model create massive leakage: meta-model learns to trust overfit base model predictions. Always use out-of-fold (OOF) predictions for stacking
  • Stacking similar models — stacking XGBoost + another XGBoost with slightly different hyperparameters provides almost no diversity. Ensemble diversity = reduced correlation between model errors. Mix model types: tree ensemble + linear model + neural network
  • "More models = better ensemble" — diminishing returns set in quickly. After 5–7 diverse models, adding more rarely helps and increases inference cost. The correlation between base model errors matters more than count
  • Using a complex meta-model — meta-models should be simple (linear regression, logistic regression, ridge). A complex meta-model on OOF predictions will overfit to the small number of base models. Ridge regression is the standard meta-learner
  • Not computing OOF predictions correctly for test set — for each CV fold, train base models on train fold, predict on val fold (these become OOF train features). For test features: average base model predictions across all k folds' versions of the model
How It's Used at Top Companies
  • Kaggle top-10 solutions: virtually all winning structured-data solutions use stacking — typically LightGBM + XGBoost + CatBoost + neural network base models with a ridge meta-learner; the ensemble typically adds 0.2–0.5% AUC over the best single model
  • Netflix Prize (2009): the winning solution was an ensemble of 107 distinct algorithms — stacking introduced the concept of learned meta-models in the competition ML community
  • Production ML at scale: full stacking is often impractical at serving time; soft voting (averaging 2–3 models) is the production-friendly alternative — simple to deploy, easy to debug, meaningful accuracy gain
  • Weather forecasting (ECMWF): ensemble forecasting is the standard in meteorology — running 50 perturbed model instances and averaging produces better probabilistic forecasts than any single deterministic model; the ML concept is borrowed from this tradition
🎯 Interview Gold

What to say: "For competitions: stack XGBoost + LightGBM + CatBoost + neural network, meta-model is ridge regression on OOF predictions — diversity across model types is the key. For production: soft voting of 2–3 models is the pragmatic choice — easy to deploy, easy to roll back. The cardinal stacking rule: meta-model must train on out-of-fold predictions, never on in-sample predictions — the leakage otherwise is severe and makes the ensemble look far better than it is on the training set."

Common mistake to avoid saying: "I averaged predictions from two XGBoost models with different seeds." — Two nearly-identical models have highly correlated errors; averaging barely helps. Diversity (different model families) is what drives ensemble gains.

Expect the follow-up: "Why does ensembling reduce variance?" — Answer: if base model errors are uncorrelated with variance σ², averaging k models reduces variance to σ²/k. In practice, errors are partially correlated, so the reduction is (1 + (k−1)ρ)·σ²/k where ρ is the average pairwise error correlation. Minimizing ρ (diversity) is equivalent to maximizing variance reduction — this is why diverse model types are critical.
