Transfer Learning & Knowledge Distillation
Two complementary strategies for leveraging existing knowledge — transfer learning adapts pre-trained representations to new tasks, while distillation compresses large model knowledge into smaller, faster, deployable ones.
Part 1 — Transfer Learning
Don't start from scratch — leverage representations learned from massive datasets and adapt them to your task. The single highest-leverage technique in applied deep learning.
The Paradigm Shift
Pre-train a large model on a massive task (ImageNet, language modeling, masked token prediction). The model learns rich, general features. Fine-tune this model on your small labeled dataset — the features transfer.
$$\begin{aligned} \text{Stage 1: Pre-train on large dataset (self-supervised or supervised)} \\ \text{Stage 2: Replace/add task head (classification, NER, etc.)} \\ \text{Stage 3: Fine-tune} \text{ --- either:} \\ \quad\text{(a) Feature extraction: freeze backbone, train only head} \\ \quad\text{(b) Full fine-tuning: unfreeze all layers, small LR} \\ \quad\text{(c) Partial fine-tuning: unfreeze last N layers} \end{aligned}$$

Parameter-Efficient Fine-Tuning (PEFT)
  • LoRA: add low-rank matrices ΔW = AB to frozen weights. Only A and B are trained (far fewer params). State-of-the-art for LLM fine-tuning.
  • Adapters: insert small bottleneck modules between frozen layers
  • Prompt tuning: learn soft prompt tokens while freezing the model
  • BitFit: only fine-tune bias terms — surprisingly effective
Catastrophic Forgetting
When fine-tuning on a new task, the model "forgets" its pre-trained knowledge. Prevention strategies:
  • Learning rate warmup: start with very small LR, increase gradually
  • Layer-wise LR decay: lower LR for earlier (more general) layers
  • Elastic weight consolidation (EWC): penalize changing weights important for old tasks
  • LoRA/Adapters: freeze base model — forgetting impossible

🎯 Interview Gold

"Transfer learning is the highest-leverage technique in applied DL. For NLP tasks I start with a pre-trained BERT/RoBERTa, add a classification head, and fine-tune. With LoRA I can fine-tune a 7B parameter LLM on a single GPU by training only 0.1% of parameters. The intuition: lower layers learn general features (edges, grammar), upper layers learn task-specific features — so for a new task in the same domain, only upper layers need to change."

Part 2 — Knowledge Distillation
Train a small "student" model to mimic a large "teacher" model — transferring knowledge without transferring parameters. The technique behind DistilBERT, TinyBERT, and most efficient production models.
The Core Idea
Instead of training student on hard labels (0 or 1), train it to match the teacher's soft probability distribution. The soft labels contain richer information — the teacher's confidence across all classes, including wrong ones.
$$\begin{aligned} \text{Teacher: large model } T \text{ with params } \theta_T \text{ (frozen during distillation)} \\ \text{Student: small model } S \text{ with params } \theta_S \text{ (trained)} \\ p_T(y|x,\tau) = \text{softmax}(z_T/\tau) \quad [\tau = \text{temperature}] \\ \mathcal{L} = \alpha\cdot\mathcal{L}_{CE}(y_{\text{hard}},p_S) + (1-\alpha)\cdot\tau^2\cdot\mathcal{L}_{KL}(p_T\|p_S) \\ \tau=1\text{: original distribution} \quad \tau>1\text{: smoother, more informative} \\ \alpha\text{: balances hard label loss and distillation loss (typically }\alpha=0.3\text{)} \end{aligned}$$
Distillation: student learns from both ground truth AND teacher's soft output distribution
Why Soft Labels Are Better
Hard label: cat = [1, 0, 0, 0]. Teacher soft label: [0.01, 0.72, 0.20, 0.07] — the teacher knows "fox" is similar to "cat". The student learns this similarity structure for free, without needing labeled examples of every pair.
📖 Brilliant Tutor

The teacher says "this is likely a fox, somewhat like a cat, unlikely a car." The student learns not just the answer but the teacher's uncertainty and reasoning — far richer signal than binary right/wrong labels.

🎯 Interview Gold

"The temperature τ is the key insight. At τ=1, softmax is sharp — nearly a one-hot. At τ=4, it's soft and reveals class similarities the model has learned. The student trained on τ=4 soft labels learns a richer representation than one trained on hard labels. In production I use distillation when I need <100ms latency but my best model is too large — DistilBERT gives 97% of BERT quality at 2× the speed, which is often the right tradeoff."
