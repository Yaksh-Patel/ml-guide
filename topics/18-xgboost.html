XGBoost — Deep Dive
The algorithm that dominated Kaggle for 5 years. Understanding it mathematically separates "I used XGBoost" from "I understand why XGBoost works."
XGBoost Objective Function
$$\begin{aligned} \mathcal{L} &= \sum_i L(y_i,\hat{y}_i) + \sum_k \Omega(f_k) \\ \Omega(f) &= \gamma T + \tfrac{1}{2}\lambda\|w\|^2 + \alpha\|w\|_1 \\ &\quad T = \text{leaves},\; w = \text{leaf weights} \\ &\quad \gamma\text{: tree complexity},\; \lambda\text{: L2},\; \alpha\text{: L1 on weights} \end{aligned}$$
This is the key innovation: regularization inside the tree structure itself. XGBoost optimizes a regularized objective, not just prediction error. Other boosting implementations lack this.
Second-Order Taylor Expansion (why it's faster)
$$\begin{aligned} L(y_i, \hat{y}_i^{(t)}) &\approx L(y_i, \hat{y}_i^{(t-1)}) + g_i f_t(x_i) + \tfrac{1}{2} h_i f_t(x_i)^2 \\ g_i &= \partial L / \partial \hat{y}_i^{(t-1)} \quad [\text{gradient}] \\ h_i &= \partial^2 L / \partial (\hat{y}_i^{(t-1)})^2 \quad [\text{hessian}] \\ w_j^* &= -\frac{\sum_{i \in j} g_i}{\sum_{i \in j} h_i + \lambda} \quad [\text{optimal leaf weight for leaf } j] \end{aligned}$$
Using the hessian (2nd derivative) means XGBoost takes curvature into account — it takes larger steps where the loss is flat and smaller steps where it curves sharply. This is why it converges faster than vanilla gradient boosting.
Split Gain Formula
$$\begin{aligned} \text{Gain} &= \frac{1}{2}\left[\frac{G_L^2}{H_L+\lambda} + \frac{G_R^2}{H_R+\lambda} - \frac{G^2}{H+\lambda}\right] - \gamma \\ G &= \text{sum of gradients in node},\quad H = \text{sum of hessians in node} \\ &\quad L/R\text{ subscripts} = \text{left/right child} \\ &\quad \text{Split is rejected if Gain} < 0 \quad\text{(this IS the pruning)} \end{aligned}$$
Missing Value Handling
XGBoost learns the default direction for missing values during training. At each split, it tries sending missing values both left and right, picks the one with higher Gain. This is why XGBoost handles NaNs natively without imputation.
System Optimizations
  • Column blocks: pre-sorted feature values stored in compressed blocks for parallel access
  • Cache-aware access: allocates internal buffers to reduce cache misses
  • Out-of-core computation: processes data larger than RAM in blocks
  • Approximate algorithm: sketch/histogram for continuous features — faster split finding

"XGBoost's edge over vanilla gradient boosting: (1) second-order Taylor expansion gives better gradient estimates and allows analytical optimal leaf weights; (2) built-in regularization (γ penalizes tree complexity, λ regularizes weights) prevents overfitting without needing post-hoc pruning; (3) approximate split finding via weighted quantile sketch scales to large datasets. My tuning strategy: fix learning_rate=0.05, n_estimators=5000 with early stopping, then tune max_depth, min_child_weight, subsample, colsample_bytree via Bayesian optimization."
