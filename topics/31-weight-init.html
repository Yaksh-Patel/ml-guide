Weight Initialization
Poor initialization causes vanishing or exploding activations before training even starts — a network that never had a chance. The right init sets up the gradient highway from epoch 0.
Why Initialization Matters
  • All zeros: symmetry problem — all neurons compute identical gradients, stay identical forever. The network never breaks symmetry.
  • All ones (or any constant): same problem — all neurons stay identical.
  • Too large: activations saturate (sigmoid/tanh), gradients vanish
  • Too small: activations shrink to zero layer by layer, gradients vanish
  • Goal: keep variance of activations approximately constant through layers
Xavier / Glorot Initialization
$$\begin{aligned} W \sim \text{Uniform}\!\left(-\sqrt{\tfrac{6}{n_{\text{in}}+n_{\text{out}}}},\; \sqrt{\tfrac{6}{n_{\text{in}}+n_{\text{out}}}}\right) \\ \text{or} \\ W \sim \mathcal{N}\!\left(0,\; \tfrac{2}{n_{\text{in}}+n_{\text{out}}}\right) \end{aligned}$$
Designed for sigmoid/tanh activations. Preserves variance in both forward and backward passes. Derived by assuming linear activations.
Sigmoid / Tanh
He / Kaiming Initialization
$$\begin{aligned} W \sim N(0, 2/n_{in}) \\ or \\ W \sim Uniform(-\sqrt{6/n_{in}}, \sqrt{6/n_{in}}) \end{aligned}$$
Designed for ReLU. Factor of 2 accounts for the half-activation (ReLU zeroes negative inputs, effectively halving the variance). Use He normal for ReLU and its variants.

Common Mistakes & Misconceptions
  • Initialising all weights to zero — symmetry breaking failure: all neurons in a layer receive identical gradients and learn identical features forever. Random initialisation (even small random values) breaks symmetry so each neuron learns a different feature
  • Using Xavier init with ReLU activations — Xavier/Glorot assumes symmetric activations (like tanh). ReLU kills ~50% of activations, halving the effective variance. Use He (Kaiming) initialisation for ReLU networks: std = √(2/fan_in) compensates for the 50% kill rate
  • "BatchNorm makes initialisation irrelevant" — BatchNorm makes training more robust to bad initialisation, but poor init can still cause NaN loss or slow convergence in the first few steps before BatchNorm stabilises. Best practice: correct init + BatchNorm together
  • Not setting LSTM forget gate bias to 1 — at initialisation, the forget gate defaults to ~0.5 (sigmoid of ~0), meaning the LSTM immediately starts forgetting past context. Setting forget gate bias=1 initialises the gate to ~0.73, helping the model retain context by default and learn to forget selectively
  • Reinitialising pretrained weights — when fine-tuning, only initialise the new task-specific head; never reinitialise the pretrained backbone. Even a "frozen" pretrained model used as a fixed feature extractor performs better than a randomly reinitialised one
How It's Used at Top Companies
  • Meta (PyTorch default): Kaiming uniform (He) is the PyTorch default for Linear and Conv2d layers — chosen as the safest general-purpose init for ReLU networks; Xavier is available for tanh/sigmoid networks via torch.nn.init.xavier_uniform_
  • Google (Transformer init): small truncated normal initialisation (std=0.02) for all transformer weights — small values prevent large attention logits at initialisation, which would cause softmax to saturate before any gradient flows
  • OpenAI (GPT residual scaling): residual projection weights scaled by 1/√(2N) where N is the number of residual layers — prevents residual stream variance from growing with depth at initialisation; used in GPT-2 and successors
  • Hugging Face (fine-tuning): pretrained transformer weights loaded as-is; only the classification head (or LoRA adapter weights) is randomly initialised — critical to never accidentally reinitialise the pretrained backbone
🎯 Interview Gold

What to say: "He (Kaiming) initialisation for ReLU networks is my default — std = √(2/fan_in) compensates for ReLU killing ~50% of activations, ensuring variance is preserved through initialisation. Xavier/Glorot for tanh/sigmoid networks. For transformers, small truncated normal (std=0.02) prevents attention logit saturation at the start. With BatchNorm, initialisation matters less because activations are re-normalised, but correct init still speeds early convergence."

Common mistake to avoid saying: "I initialised all weights to zero to start from a neutral state." — This causes complete symmetry: every neuron receives the same gradient and learns the same feature forever. Random initialisation is not about starting at a "good" point — it's about breaking symmetry so neurons can specialise.

Expect the follow-up: "Why do we care about variance preservation in initialisation?" — Answer: if variance grows through layers, activations explode; if it shrinks, activations vanish to zero and gradients disappear. He and Xavier init are designed so that the variance of activations equals the variance of inputs at each layer in expectation — keeping signals in a sensible range throughout the forward and backward passes.
