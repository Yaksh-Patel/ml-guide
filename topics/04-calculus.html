Calculus & Optimization
The engine behind every learning algorithm — gradients tell the model which direction to nudge its parameters to reduce error. Without this, gradient descent is magic. With it, it's engineering.
Derivatives
$$\begin{aligned} f'(x) = \lim_{h \to 0} (f(x+h)-f(x))/h \\ \frac{d}{dx}(x^{n}) = nx^{n-1} \\ \frac{d}{dx}(e^x) = e^x \\ \frac{d}{dx}(\ln x) = 1/x \\ \frac{d}{dx}(\sigma (x)) = \sigma (x)(1-\sigma (x)) \; (\text{\text{sigmoid} gradient}) \\ \frac{d}{dx}(\text{ReLU}(x)) = \begin{cases}1 & x > 0 \\ 0 & \text{otherwise}\end{cases} \end{aligned}$$
Gradient (Multivariable)
$$\begin{aligned} \nabla f = \left[\frac{\partial f}{\partial x_1},\, \frac{\partial f}{\partial x_2},\, \ldots,\, \frac{\partial f}{\partial x_n}\right]^T \quad \text{(points in direction of steepest ascent)} \\ \text{Gradient descent: } \theta \leftarrow \theta - \alpha \nabla L(\theta) \\ \text{Partial derivative: hold all other vars constant} \end{aligned}$$
The gradient is a vector field — at each point it shows the direction to climb fastest. Gradient descent walks opposite to this, descending toward a minimum.
Chain Rule — The Heart of Backpropagation
If L = f(g(h(x))), then dL/dx = (dL/df)(df/dg)(dg/dh)(dh/dx) — multiply local gradients along the path.
$$\partial L/\partial W^1 = (\partial L/\partial \hat{y})\cdot(\partial \hat{y}/\partial h^2)\cdot(\partial h^2/\partial h^1)\cdot(\partial h^1/\partial W^1)$$
Chain Rule: gradient flows backward
Convexity
  • Convex function: any local min = global min. Logistic regression loss is convex.
  • Non-convex: neural networks — many local minima, but in practice most are good enough
  • Saddle points: gradient = 0 but not a min — SGD noise helps escape these
Useful Calculus Tricks
  • Log-trick: max P(data) ≡ max log P(data) — turns products into sums
  • Jensen's inequality: E[f(X)] ≥ f(E[X]) for convex f — used in ELBO
  • Hessian (∂²L/∂θ²): matrix of 2nd derivatives — curvature, used in Newton's method
Common Mistakes & Misconceptions
  • Chain rule direction confusion in backprop — the chain rule flows backward: ∂L/∂w = (∂L/∂a) × (∂a/∂z) × (∂z/∂w). A common mistake is computing partial derivatives in the wrong order or forgetting to multiply by the upstream gradient. Always trace the computation graph explicitly when debugging gradient shapes
  • Gradient vs derivative conflation — the derivative is a scalar (for scalar outputs); the gradient is a vector (partial derivatives w.r.t. each input). The Jacobian is a matrix of partial derivatives (for vector-to-vector functions). Neural networks compute Jacobians at each layer; understanding the difference matters for debugging shape errors
  • "Saddle points trap gradient descent in neural networks" — early theory suggested saddle points were the main difficulty. Empirically, SGD with noise and momentum escapes saddle points readily. The real challenge is flat loss landscapes (poorly conditioned Hessians) and sharp minima that don't generalise well
  • Forgetting the log-sum-exp trick for numerical stability — computing log(sum(exp(x))) directly overflows for large x (exp(1000) = ∞). The log-sum-exp trick: log(Σ exp(xᵢ)) = c + log(Σ exp(xᵢ - c)) where c = max(x). This is how softmax is implemented stably in every framework
  • Second-order methods are always better than first-order — Newton's method uses the Hessian (second derivatives) and converges in far fewer steps than gradient descent. But computing and inverting the full Hessian is O(n³) for n parameters — infeasible for neural networks with millions of parameters. This is why Adam (a diagonal Hessian approximation) is practical and full second-order methods are not
How It's Used at Top Companies
  • Google Brain / DeepMind (optimizer research): Shampoo and SOAP are practical second-order optimisers that approximate the Hessian using Kronecker-factored curvature; used in LLM training to achieve faster convergence than Adam with manageable memory overhead
  • Meta (automatic differentiation — torch.autograd): PyTorch's autograd engine records a dynamic computation graph during the forward pass; calling .backward() traverses this graph in reverse applying the chain rule at each node — calculus implemented as software
  • OpenAI (gradient checkpointing at scale): trading recomputation for memory — during backprop, only a subset of activations are stored; the rest are recomputed from checkpoints. The mathematical cost analysis (33% extra FLOPs for ~10× memory reduction) is a direct calculus-in-practice application
  • Physics-informed ML: Physics-Informed Neural Networks (PINNs) incorporate PDE residuals as loss terms — ∂u/∂t + N[u] = 0 must hold throughout the domain. The loss gradient w.r.t. neural net parameters involves automatic differentiation through both the network and the PDE residual
🎯 Interview Gold

What to say: "Every training step is applied calculus: compute the gradient of loss w.r.t. every parameter via the chain rule (backprop), then step in the negative gradient direction (gradient descent). Understanding this lets you debug vanishing gradients — if activations saturate (sigmoid at extremes), local derivatives → 0, gradients vanish, early layers stop learning. ReLU's derivative is exactly 1 for positive inputs, so gradients flow unchanged — that's why it fixed vanishing gradients. The log-sum-exp trick is calculus in practice: numerical stability for softmax and cross-entropy at scale."

Common mistake to avoid saying: "Gradient descent finds the global minimum." — Only for convex loss surfaces (logistic regression, linear regression). Neural network losses are highly non-convex; gradient descent finds a local minimum. In practice this is fine because most local minima in overparameterised networks are close to the global minimum — but this is an empirical observation, not a guarantee.

Expect the follow-up: "What is the difference between the gradient and the Jacobian?" — Answer: the gradient ∇f is the vector of partial derivatives of a scalar-valued function f: ℝⁿ → ℝ, shape (n,). The Jacobian J is the matrix of partial derivatives of a vector-valued function f: ℝⁿ → ℝᵐ, shape (m, n). In neural networks, each layer maps a vector to a vector — its local derivative is a Jacobian. Backprop computes the product of Jacobians from output to input via the chain rule, but never materialises the full Jacobian matrix (too large); it uses vector-Jacobian products (VJPs) instead.
