ML Paradigms
How you frame the problem determines everything — the algorithm family, the data you need, the evaluation strategy, and the deployment architecture.
Supervised Learning
Learn a mapping f: X → y from labeled (input, output) pairs. The label is the signal.
  • Classification: y is categorical (spam/not-spam, credit default)
  • Regression: y is continuous (house price, revenue)
  • Examples: logistic regression, decision trees, XGBoost, neural networks
  • Key risk: label quality — garbage labels = garbage model
Unsupervised Learning
Find structure in data without labels. No y — just X.
  • Clustering: K-Means, DBSCAN, hierarchical — group similar points
  • Dim reduction: PCA, t-SNE, UMAP — compress feature space
  • Density estimation: GMMs, KDEs — model P(x)
  • Anomaly detection: isolation forest, autoencoders
Semi-Supervised Learning
Tiny labeled set + large unlabeled set. Labels are expensive; data is cheap.
  • Self-training: label unlabeled data with your own model, retrain
  • Pseudo-labels + consistency regularization
  • BERT pretraining is semi-supervised (MLM on unlabeled text)
Reinforcement Learning
Agent takes actions in an environment, receives reward signals, learns a policy π: state → action to maximize cumulative reward.
  • No fixed dataset — agent generates its own data through interaction
  • Key challenge: credit assignment & exploration vs exploitation
  • Used in RLHF (aligning LLMs), game playing, robotics, trading
Self-Supervised Learning (the modern paradigm)
Create labels from the data itself — no human annotation needed. The model learns rich representations as a side-effect of solving a pretext task.

Common Mistakes & Misconceptions
  • "Unsupervised learning is just clustering" — unsupervised encompasses clustering (K-Means, DBSCAN), dimensionality reduction (PCA, UMAP), density estimation (GMM, KDE), generative modelling (VAE, GAN, diffusion), and anomaly detection. Clustering is one small subfield
  • Treating self-supervised and unsupervised as synonymous — self-supervised learning generates labels from the data structure itself (predict masked tokens, predict next frame, predict image rotation). The labels are automatic, not manually annotated. Unsupervised learning has no labels at all (clustering, PCA). Self-supervised typically produces better representations than unsupervised methods
  • "More data always means supervised learning wins" — not when labels are expensive, noisy, or unavailable. Semi-supervised learning (few labels + much unlabeled data) and self-supervised pretraining + fine-tuning often outperform supervised-only approaches with limited labelled data. Label efficiency is a central concern in applied ML
  • Applying reinforcement learning where supervised learning suffices — RL requires a reward signal, an environment, and many trial-and-error interactions — extremely sample-inefficient. If you can collect (input, desired output) pairs, supervised learning is almost always simpler, faster, and more reliable. RL is for sequential decision problems where the correct output can't be specified directly (game playing, robot control, LLM alignment)
  • Ignoring the distribution shift between pretraining and fine-tuning — a model pretrained on web text and fine-tuned on medical notes faces distribution shift at both the vocabulary and semantic level. Domain-adaptive pretraining (continue MLM on domain corpus before task fine-tuning) bridges this gap and consistently improves downstream task performance
How It's Used at Top Companies
  • OpenAI / Anthropic (self-supervised pretraining): next-token prediction on hundreds of billions of tokens — a self-supervised task that requires no human labels but forces the model to learn syntax, semantics, factual knowledge, and reasoning as side effects of predicting the next word; the foundation of all frontier LLMs
  • Google (semi-supervised vision — FixMatch / SimCLR): ImageNet pre-training is fully self-supervised (contrastive views); fine-tuning on 1% of labelled ImageNet achieves near-full supervised performance — dramatic label efficiency gains from the pretrained representation
  • DeepMind (RL — AlphaFold, AlphaCode): reinforcement learning from self-play used in AlphaGo/AlphaZero; RLHF (a hybrid) used in Gemini alignment; pure RL for protein folding via energy function reward. RL applied only where sequential decisions and delayed rewards make supervised approaches insufficient
  • Tesla (Autopilot — weakly supervised): billions of miles of driving video with weak labels (auto-annotated by fleet data, not human review) used to train perception models; human annotation reserved only for the hardest edge cases; semi-supervised at automotive scale
🎯 Interview Gold

What to say: "For a new problem I start with: do I have labels? If yes — supervised. If no — unsupervised or self-supervised. The most powerful paradigm today is self-supervised pretraining + fine-tuning: use massive unlabelled data to learn general representations (next-token prediction, masked image modelling), then fine-tune on a small labelled set. This is why BERT and GPT dominate NLP despite labelled data being scarce — they leverage internet-scale unlabelled data that supervised approaches can't exploit."

Common mistake to avoid saying: "I'd use reinforcement learning for this recommendation problem." — RL is sample-inefficient and complex to productionise. Unless the problem is explicitly sequential with delayed rewards (multi-step user journey optimisation), supervised or bandit approaches are simpler and more reliable. Reach for RL only when you can't define a direct supervised signal.

Expect the follow-up: "What's the difference between online and offline reinforcement learning?" — Answer: online RL collects experience by interacting with the environment during training (AlphaGo, RLHF with PPO rollouts) — expensive but can explore to improve. Offline RL (also called batch RL) trains entirely from a fixed logged dataset of past interactions (e.g., historical user trajectories) without further environment interaction — safer for production but can't explore beyond the data distribution. DPO is effectively offline RL for LLM alignment.
