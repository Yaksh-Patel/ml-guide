ML Paradigms
How you frame the problem determines everything — the algorithm family, the data you need, the evaluation strategy, and the deployment architecture.
Supervised Learning
Learn a mapping f: X → y from labeled (input, output) pairs. The label is the signal.
  • Classification: y is categorical (spam/not-spam, credit default)
  • Regression: y is continuous (house price, revenue)
  • Examples: logistic regression, decision trees, XGBoost, neural networks
  • Key risk: label quality — garbage labels = garbage model
Unsupervised Learning
Find structure in data without labels. No y — just X.
  • Clustering: K-Means, DBSCAN, hierarchical — group similar points
  • Dim reduction: PCA, t-SNE, UMAP — compress feature space
  • Density estimation: GMMs, KDEs — model P(x)
  • Anomaly detection: isolation forest, autoencoders
Semi-Supervised Learning
Tiny labeled set + large unlabeled set. Labels are expensive; data is cheap.
  • Self-training: label unlabeled data with your own model, retrain
  • Pseudo-labels + consistency regularization
  • BERT pretraining is semi-supervised (MLM on unlabeled text)
Reinforcement Learning
Agent takes actions in an environment, receives reward signals, learns a policy π: state → action to maximize cumulative reward.
  • No fixed dataset — agent generates its own data through interaction
  • Key challenge: credit assignment & exploration vs exploitation
  • Used in RLHF (aligning LLMs), game playing, robotics, trading
Self-Supervised Learning (the modern paradigm)
Create labels from the data itself — no human annotation needed. The model learns rich representations as a side-effect of solving a pretext task.

🎯 Interview Gold

"For a new problem, I first ask: do I have labels? If yes, supervised. If not, unsupervised or self-supervised. The most powerful paradigm today is self-supervised pretraining + fine-tuning — use massive unlabeled data to learn representations, then fine-tune on a small labeled set. This is why BERT and GPT-style models dominate NLP despite labeled data being scarce."
