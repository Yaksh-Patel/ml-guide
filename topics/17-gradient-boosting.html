Gradient Boosting
Sequential ensemble where each tree corrects the residual errors of all previous trees. Reduces bias (unlike Random Forest which reduces variance). The dominant algorithm for structured data competitions.
Core Algorithm
$$\begin{aligned} F_0(x) = \text{base prediction (e.g., mean of } y) \\ r_{im} = -\frac{\partial L(y_i, F_{m-1}(x_i))}{\partial F_{m-1}(x_i)} \quad \text{[pseudo-residuals]} \\ \text{Fit tree } h_m \text{ to pseudo-residuals } \{r_{im}\} \\ F_m(x) = F_{m-1}(x) + \eta \cdot h_m(x) \\ F(x) = F_0 + \eta \cdot \sum_m h_m(x) \end{aligned}$$
For squared loss: pseudo-residuals = actual − predicted (simple residuals). For log-loss: pseudo-residuals = y − σ(F(x)). Each tree is a weak learner (shallow, high bias) trained on the gradient direction of the loss.
XGBoost
  • Second-order Taylor expansion for faster convergence
  • Built-in L1/L2 regularization on leaf weights
  • Handles missing values natively (learns default direction)
  • Parallel tree construction per level
Gold standard
LightGBM
  • Leaf-wise growth (vs level-wise in XGBoost) → deeper, more accurate
  • GOSS: only large-gradient samples for split finding
  • EFB: bundles mutually exclusive features
  • 10-20x faster than XGBoost on large data
Speed king
CatBoost
  • Native categorical encoding (ordered target encoding)
  • Ordered boosting prevents target leakage
  • Symmetric trees → fast inference
  • Minimal hyperparameter tuning needed
Best for cats

📖 The Consulting Analogy

XGBoost hires 500 consultants sequentially. Consultant 1 gives a report. Consultant 2 reads report 1 and specifically studies its mistakes, then corrects them. Consultant 3 corrects what 1 and 2 missed together. Each expert focuses only on the residual errors of all predecessors. By the end, the team has collectively covered every blind spot. Too many rounds? The last consultants start memorizing trivia (overfitting) — early stopping saves you.

🎯 Interview Gold

"Gradient boosting reduces bias via sequential error correction — each tree fits the negative gradient of the loss. Unlike bagging (reduces variance), boosting reduces bias. The learning rate η is a shrinkage factor that prevents overfitting. I always use early stopping with a validation set (eval_metric + early_stopping_rounds in XGBoost). For most tabular problems, LightGBM is my first choice — faster, matches XGBoost performance, and handles large datasets better."
