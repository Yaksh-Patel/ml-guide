RNNs & LSTMs
Neural networks with memory — process sequences step by step, maintaining a hidden state that summarizes history. Pre-Transformer architecture for sequential data.
Vanilla RNN
$$\begin{aligned} h_t = \tanh(W_{hh}h_{t-1} + W_{xh}x_t + b) \\ y_t = W_{hy}h_t + b_y \end{aligned}$$
The same weights (Wₕₕ, Wₓₕ) are applied at every time step — "weight sharing across time" analogous to CNN's weight sharing across space. Problem: for long sequences, multiplying Wₕₕᵀ many times → vanishing or exploding gradients.
RNN Unrolled — and LSTM gates controlling information flow
LSTM — Long Short-Term Memory
$$\begin{aligned} f_t = \sigma(W_f[h_{t-1}, x_t] + b_f) \quad \text{(forget gate)} \\ i_t = \sigma(W_i[h_{t-1}, x_t] + b_i) \quad \text{(input gate)} \\ \tilde{g}_t = \tanh(W_g[h_{t-1}, x_t] + b_g) \quad \text{(candidate memory)} \\ C_t = f_t \odot C_{t-1} + i_t \odot \tilde{g}_t \quad \text{(cell state update)} \\ o_t = \sigma(W_o[h_{t-1}, x_t] + b_o) \quad \text{(output gate)} \\ h_t = o_t \odot \tanh(C_t) \quad \text{(hidden state)} \end{aligned}$$
The cell state Cₜ is a conveyor belt — it carries information across many time steps with only multiplicative (not additive) interactions, preventing gradient vanishing. Gates (sigmoid squashed → [0,1]) control: forget (what to erase), input (what to add), output (what to expose).
GRU — Gated Recurrent Unit
$$\begin{aligned} z_t = \sigma(W_z[h_{t-1}, x_t]) \quad \text{(update gate)} \\ r_t = \sigma(W_r[h_{t-1}, x_t]) \quad \text{(reset gate)} \\ \tilde{h}_t = \tanh(W[r_t \odot h_{t-1},\, x_t]) \\ h_t = (1-z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t \end{aligned}$$
2 gates instead of 3. No separate cell state. Fewer parameters than LSTM — often matches performance, trains faster.
Bidirectional RNN
Run one RNN forward (x₁→xₙ) and one backward (xₙ→x₁). Concatenate hidden states. Each position sees both past AND future context.
$$\bar{h}_t = \text{concat}(\text{RNN}_{\text{forward}}(x_t),\; \text{RNN}_{\text{backward}}(x_t))$$
Used in BERT's architecture conceptually. Requires full sequence — can't stream. More powerful than unidirectional for NLU tasks.
📖 Goldfish vs Elephant Analogy

Vanilla RNN is a goldfish — forgets what happened 10 steps ago because the information has to pass through 10 matrix multiplications. LSTM is an elephant with a notebook (cell state) — it decides what to write, what to erase, and what to read. The cell state highway carries memories intact across hundreds of steps.

Common Mistakes & Misconceptions
  • "LSTMs solved the vanishing gradient problem" — LSTMs greatly mitigate vanishing gradients via the cell state highway (additive gradient flow), but not fully. For sequences >500 steps, LSTMs still struggle. Transformers with attention handle long-range dependencies fundamentally better
  • Using a vanilla RNN instead of LSTM/GRU — vanilla RNNs suffer severe vanishing gradients on sequences longer than ~10 steps. Always use LSTM or GRU for sequence modelling; vanilla RNNs are only for teaching the concept
  • Not setting forget gate bias = 1 — PyTorch LSTM default forget gate bias is 0 (gate sigmoid(0) ≈ 0.5 — immediately starts forgetting). Setting forget gate bias=1 initialises the gate to ~0.73, encouraging the model to retain context early in training
  • Using bidirectional LSTMs for autoregressive generation — bidirectional LSTMs see the full sequence in both directions; they cannot generate tokens left-to-right because they require the future to process the past. Use unidirectional LSTMs for generation; bidirectional only for encoding/classification where the full sequence is available
  • Not clipping gradients for RNN training — RNNs are especially prone to exploding gradients due to repeated matrix multiplication through time. Always apply gradient clipping (clip_grad_norm=1.0 or 5.0) when training RNNs/LSTMs
How It's Used at Top Companies
  • Google (on-device speech — RNN-T): streaming speech recognition uses RNN Transducer (RNN-T) — unidirectional LSTM encoder processes audio frames as they arrive, enabling real-time transcription without seeing future audio; Transformers can't do this without modification
  • Spotify (music generation — early): LSTM-based melody generation models before transformers dominated; still used in resource-constrained mobile generation tasks where transformer inference is too slow
  • Financial time series: stacked LSTMs for stock return prediction and anomaly detection in transaction streams; advantage over Transformers: O(n) inference time vs O(n²) attention — critical for high-frequency streaming data
  • Manufacturing / IoT (edge devices): GRUs (simpler than LSTMs, fewer parameters) deployed on microcontrollers for real-time sensor anomaly detection — GRU's smaller memory footprint fits in KB-scale device RAM
🎯 Interview Gold

What to say: "LSTMs solve the vanishing gradient problem of vanilla RNNs via the cell state — an additive gradient highway that carries information across hundreds of steps without multiplicative decay. Three gates: forget (what to erase from cell state), input (what new info to write), output (what to read to hidden state). Today, for most NLP tasks I default to Transformer-based architectures — better long-range dependencies, parallelisable training. LSTMs remain valuable for: streaming/online processing (no future context needed), time series, and resource-constrained edge deployment."

Common mistake to avoid saying: "LSTMs completely solved vanishing gradients." — They greatly mitigate it but don't eliminate it. For very long sequences (>500 steps), LSTMs still degrade. Transformers with attention are architecturally superior for long-range dependencies.

Expect the follow-up: "What's the difference between LSTM and GRU?" — Answer: GRU merges the forget and input gates into a single update gate and combines cell and hidden state. Result: fewer parameters (~75% of LSTM), faster training, often comparable performance. GRU is preferred for smaller datasets or constrained compute; LSTM is preferred for very long sequences where the extra gate expressivity helps.
