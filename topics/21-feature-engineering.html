Feature Engineering & Data Processing
The art and science of turning raw data into signals that algorithms can use — arguably the highest-ROI activity in applied ML. Better features beat better algorithms every time.
Missing Value Strategies
MethodWhen to Use
Drop rowsOnly if <5% missing and random (MCAR)
Mean/median imputationMCAR; doesn't preserve variance
Mode imputationCategorical features
KNN imputationMAR; uses similar rows; expensive
Model imputationBest accuracy; risk of leakage
Missing indicator + imputeWhen missingness itself is signal
XGBoost/LightGBMHandle NaN natively — no imputation needed
⚠️ Types of Missingness

MCAR (completely at random): safe to drop. MAR (at random given other vars): impute. MNAR (not at random — the value is missing because of what it is): add missingness indicator as feature!

Feature Scaling
Standardization (Z-score)
$$x' = (x - \mu) / \sigma \rightarrow N(0,1)$$
Use for: linear models, SVM, PCA, neural networks. Does NOT bound to [0,1] — outliers still present.
Min-Max Normalization
$$x' = (x - xmin) / (xmax - xmin) \rightarrow \quad \text{[0,1]}$$
Use for: neural networks when you need bounded input. Sensitive to outliers — one extreme value squashes everything else.
Never scale: tree-based models (XGBoost, RF, decision trees) — splits are rank-based, scale doesn't matter. Always scale: KNN, SVM, linear/logistic, neural networks, PCA.

🎯 Interview Gold

"Feature engineering is where I spend most of my time in real ML projects. For credit risk: payment ratio = payments_made / total_due is more predictive than either raw number. For e-commerce: days_since_last_purchase + log(purchase_frequency) + interaction terms often dominate raw features. I use SHAP for feature selection — it's unbiased and shows both direction and magnitude of impact. Target encoding with 5-fold CV-in-fold is the best technique for high-cardinality categories but leakage is the #1 risk."
