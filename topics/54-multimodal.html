Multimodal Models
Multimodal models process and connect information across multiple modalities — text, images, audio, video, and structured data. They are one of the fastest-growing areas of applied ML: powering image search, visual question answering, document understanding, generative image editing, and the vision capabilities now standard in frontier LLMs.
Why Multimodal Matters
  • The real world is multimodal — products have images AND text descriptions; medical records have scans AND notes; videos have frames AND transcripts
  • Text-only models miss visual signals that are often more informative (e.g., a skin lesion image conveys more than its textual description)
  • Vision capabilities are now standard in frontier LLMs (GPT-4V, Claude 3+, Gemini) — understanding how they work is increasingly expected in ML interviews
  • Multimodal embeddings unlock new applications: image search via text query, cross-modal retrieval, visual grounding, and rich product/content understanding

Part 1 — CLIP: Contrastive Language-Image Pretraining
CLIP (Radford et al., OpenAI 2021) — The Foundation
CLIP trains two encoders — image and text — to produce embeddings in a shared vector space, using contrastive learning on 400M image-caption pairs scraped from the internet. Matching pairs are pushed together; non-matching pairs are pushed apart.
$$\begin{aligned} \mathbf{I}_i &= f_{\text{image}}(\text{image}_i) \in \mathbb{R}^d \quad \text{(image encoder, e.g. ViT-L)} \\ \mathbf{T}_i &= f_{\text{text}}(\text{caption}_i) \in \mathbb{R}^d \quad \text{(text encoder, e.g. Transformer)} \\ \text{Contrastive loss (InfoNCE):} \\ \mathcal{L} &= -\frac{1}{N}\sum_{i=1}^{N}\left[\log\frac{\exp(\mathbf{I}_i \cdot \mathbf{T}_i / \tau)}{\sum_{j=1}^{N}\exp(\mathbf{I}_i \cdot \mathbf{T}_j / \tau)}\right. \\ &\quad\quad\quad\quad\quad + \left.\log\frac{\exp(\mathbf{T}_i \cdot \mathbf{I}_i / \tau)}{\sum_{j=1}^{N}\exp(\mathbf{T}_j \cdot \mathbf{I}_i / \tau)}\right] \\ \tau &= \text{learned temperature parameter} \end{aligned}$$
How the contrastive loss works
  • In a batch of N image-caption pairs, there are N correct pairings and N²-N incorrect pairings
  • For each image: maximise similarity to its correct caption vs all other captions in the batch (and vice versa)
  • Temperature τ controls sharpness of the distribution — lower τ = sharper contrast
  • With N=32,768 (large batch): each image has 32,767 negatives — a very hard contrastive problem that forces rich representations

Zero-shot classification with CLIP

CLIP's shared embedding space enables zero-shot image classification: describe class labels as text ("a photo of a dog", "a photo of a cat"), compute text embeddings, find the label whose text embedding is most similar to the image embedding. No task-specific training needed — CLIP achieves competitive accuracy on ImageNet zero-shot.

Vision Transformer (ViT) — How Images Become Tokens
CLIP's image encoder (and most modern vision models) uses a Vision Transformer — treating an image as a sequence of patches, just like words in a sentence.
$$\begin{aligned} \text{Image } &\in \mathbb{R}^{H \times W \times C} \xrightarrow{\text{split}} N \text{ patches of size } P \times P \\ N &= \frac{HW}{P^2} \quad \text{(e.g. 224×224 image, P=16 → 196 patches)} \\ \text{Patch}_i &\xrightarrow{\text{linear proj}} \mathbf{e}_i \in \mathbb{R}^d \quad \text{(patch embedding)} \\ \text{Input} &= [\mathbf{e}_{\text{CLS}},\ \mathbf{e}_1 + \mathbf{p}_1,\ \mathbf{e}_2 + \mathbf{p}_2,\ \ldots,\ \mathbf{e}_N + \mathbf{p}_N] \end{aligned}$$
  • [CLS] token: prepended learnable token; its final embedding = the image representation (same pattern as BERT)
  • Position embeddings: 1D learned positions for each patch — ViT doesn't inherently know patch spatial relationships
  • Full Transformer attention over all patches — each patch can attend to every other patch regardless of spatial distance
  • Variants: ViT-B/16 (86M params), ViT-L/14 (307M, used by CLIP), ViT-H/14 (632M) — B/L/H = Base/Large/Huge
Part 2 — Vision-Language Models (VLMs)
LLaVA — Large Language and Vision Assistant
LLaVA (Liu et al., 2023) connects a vision encoder (CLIP ViT) to an LLM (Llama/Vicuna) via a simple learned projection layer, enabling natural language conversations about images. It established the dominant architecture pattern for open-source VLMs.
Architecture
  • Vision encoder: CLIP ViT-L/14 — produces 256 image patch embeddings
  • Projection layer (MLP): maps 1024-dim CLIP embeddings → 4096-dim LLM token space. This is the bridge between modalities
  • Language model: Llama / Mistral / Qwen — receives the projected image tokens as if they were text tokens, then auto-regressively generates the response
  • Image tokens are inserted at a special <image> position in the prompt; LLM attends to them via its standard attention mechanism
Training stages
StageWhat trainsData
1. Pre-trainingProjection layer only558K image-caption pairs; teaches vision-language alignment
2. Fine-tuningProjection + LLM (LoRA or full)158K visual instruction-following pairs (LLaVA-Instruct); teaches following instructions about images
Key insight: CLIP and the LLM are both frozen or lightly fine-tuned; only the small projection layer is trained from scratch in stage 1. This makes training cheap — LLaVA-7B can be trained in ~1 day on 8× A100s.

Part 3 — Multimodal Embeddings & Applications
Cross-Modal Embedding Applications
Image Search via Text
  • Index product images as CLIP embeddings in a vector DB
  • At query time: embed the text query → ANN search over image embeddings
  • Returns visually matching products for natural language queries: "red floral summer dress"
  • No keyword matching — pure semantic visual similarity
  • Used by: Pinterest Lens, Google Shopping, Shopify, ASOS
Image-to-Image Search
  • Embed query image → find visually similar images in the index
  • CLIP embedding captures semantic similarity (not just pixel similarity)
  • "Find more like this" product recommendation
  • Duplicate and near-duplicate detection at scale
  • Visual plagiarism detection; stock photo search
Document Understanding
  • VLMs can read text within images (OCR), understand charts/tables, and answer questions about PDF pages
  • Replaces fragile PDF parsing pipelines that break on complex layouts
  • GPT-4V / Claude 3 can extract structured data from invoices, contracts, receipts
  • Used by: DocuSign, Harvey AI (legal docs), financial document processing
Fine-Tuning CLIP for Domain-Specific Tasks
Pre-trained CLIP is trained on internet images — it may perform poorly on medical images, satellite imagery, or industrial defect detection where training data distribution is very different. Domain-specific fine-tuning can dramatically improve performance.
import torch
from transformers import CLIPModel, CLIPProcessor
from torch.optim import AdamW

model = CLIPModel.from_pretrained("openai/clip-vit-large-patch14")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-large-patch14")

# Fine-tune on domain-specific image-text pairs
# e.g. pathology images + radiology report captions
optimizer = AdamW(model.parameters(), lr=1e-5)

for batch in domain_dataloader:
    inputs = processor(
        text=batch["captions"],
        images=batch["images"],
        return_tensors="pt",
        padding=True
    )
    outputs = model(**inputs,
                    return_loss=True)
    loss = outputs.loss  # contrastive loss
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()
Alternatives to full fine-tuning
  • Linear probe: freeze CLIP, train a linear classifier on top of image embeddings. Fast baseline — often surprisingly good
  • LoRA on CLIP: add LoRA adapters to the vision transformer; train adapters on domain data while keeping base CLIP frozen. 10–30× fewer trainable params than full fine-tuning
  • Prompt tuning: learn a soft prefix for text descriptions ("a dermoscopy image of [class]") — minimal compute, competitive performance
  • Domain-specific CLIP variants: BioMedCLIP (medical), RemoteCLIP (satellite), FashionCLIP (retail) — pre-trained on relevant domain data
Part 4 — Generative Multimodal Models
Text-to-Image Generation — Architecture Overview
Text-to-image models (Stable Diffusion, DALL-E 3, Midjourney) generate images conditioned on text prompts by combining a text encoder (CLIP or T5) with a diffusion model or autoregressive image decoder.
Latent Diffusion (Stable Diffusion)
  • VAE encoder: compresses image from pixel space to a compact latent space (8× smaller) — diffusion operates here
  • Text encoder: CLIP or T5 encodes the text prompt → conditioning signal
  • U-Net / DiT: iteratively denoises a random Gaussian latent, guided by the text embedding at each step
  • CFG (Classifier-Free Guidance): run U-Net twice (with and without conditioning); push the result towards the conditioned direction. Guidance scale controls creativity vs fidelity
  • VAE decoder: maps denoised latent back to pixel space

Common Mistakes & Misconceptions
  • Using CLIP embeddings for text-only retrieval — CLIP's text encoder is optimised for vision-language alignment, not pure text semantic similarity. Use a dedicated text embedding model (BGE, E5) for text-only retrieval
  • Assuming VLMs "see" images like humans — VLMs process a fixed grid of patches with no explicit spatial reasoning. They can struggle with counting, spatial relationships ("is the red ball to the left or right of the blue cube?"), and fine-grained differences between similar images
  • Fine-tuning on tiny domain datasets without regularisation — CLIP's general knowledge is its strength; aggressive fine-tuning on small domain data causes catastrophic forgetting. Use LoRA or linear probing first
  • Ignoring VLM hallucination — VLMs confidently describe objects not present in an image, especially when the prompt implies they should be there. Always evaluate hallucination rate (POPE benchmark) for production deployments
  • Using CLIP patch size too large for fine-grained tasks — ViT-L/14 (14px patches, 256 patches for 224×224 image) has more spatial resolution than ViT-L/32 (49 patches). For tasks requiring fine-grained visual understanding (medical imaging, defect detection), use the smaller patch size

  • Pinterest: CLIP-based visual search — "Shop the Look" uses image embeddings to find visually similar shoppable products from any pinned image
  • Shopify / Amazon: CLIP embeddings for product image similarity — "customers also viewed" powered by visual similarity, not just collaborative filtering

🎯 Interview Gold

How does CLIP work? "CLIP trains a vision encoder (ViT) and a text encoder (Transformer) jointly using contrastive learning on 400M image-caption pairs. Matching pairs are pushed together in the embedding space; non-matching pairs are pushed apart. This creates a shared representation space where similar concepts — regardless of modality — end up close together. The result: zero-shot image classification, image-text retrieval, and semantic image search without task-specific training."

How does a VLM like LLaVA work? "LLaVA connects a CLIP vision encoder to an LLM via a small learned projection layer. The image is split into patches, encoded by ViT into 256 patch embeddings, projected into the LLM's embedding space, and then concatenated with the text prompt tokens. The LLM sees image tokens and text tokens together and generates a response auto-regressively — the same way it would for a text-only prompt. Only the projection layer is trained from scratch; CLIP and the LLM are fine-tuned lightly or frozen."

Multimodal retrieval in production? "Embed all images with CLIP and index in a vector DB. At query time, embed the text query with CLIP's text encoder — the shared embedding space means the query vector is directly comparable to image vectors. Run ANN search to find the top-k visually matching images. This is how Pinterest Lens, Google Shopping, and most e-commerce visual search works. CLIP's zero-shot generalisation means you don't need to label training data for each new category — the pre-trained space already understands 'red floral dress' vs 'blue striped shirt'."
