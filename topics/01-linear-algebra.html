Linear Algebra
The language of ML — every dataset is a matrix, every model is a transformation, every distance is a norm. Master this and the rest of ML becomes legible.
Vectors
An ordered list of numbers. A single data point is a vector. Carries both direction and magnitude.
$$\begin{aligned} \mathbf{v} = [v_1, v_2, \ldots, v_n]^T \quad \text{(column vector)} \\ \|\mathbf{v}\|_2 = \sqrt{v_1^2 + v_2^2 + \cdots + v_n^2} \quad \text{(L2 norm / length)} \\ \mathbf{u} \cdot \mathbf{v} = \sum_i u_i v_i = |\mathbf{u}||\mathbf{v}|\cos\theta \quad \text{(dot product)} \end{aligned}$$
  • Dot product = 0: vectors are perpendicular
  • cos θ close to 1: vectors point in same direction (used in similarity search)
Matrices
A 2D grid of numbers. Your dataset X is an (n×d) matrix — n rows = samples, d cols = features.
$$\begin{aligned} A \in \mathbb{R}^{n \times d} \\ (AB)_{i,j} = \sum_k A_{i,k} B_{k,j} \quad \text{(inner dims must match)} \\ A^T \text{ transposes rows} \leftrightarrow \text{columns} \\ A^{-1} \text{: exists iff } \det(A) \neq 0 \text{ (square, full rank)} \end{aligned}$$
Eigenvalues & Eigenvectors
For square matrix A: Av = λv — the matrix only scales vector v (doesn't rotate it). λ is the eigenvalue (scale factor), v is the eigenvector.
$$\det(A - \lambda I) = 0 \;\rightarrow\; \text{solve for } \lambda \;\rightarrow\; \text{then solve } (A - \lambda I)\mathbf{v} = 0 \text{ for } \mathbf{v}$$
  • PCA uses this: eigenvectors of covariance matrix = principal components
  • Largest eigenvalue: direction of maximum variance
  • Sum of eigenvalues = trace(A); Product = det(A)
Eigenvector: only scaled, not rotated
SVD — Singular Value Decomposition
$$\begin{aligned} A = U \Sigma V^T \\ U\text{: left singular vectors } (m \times m) \\ \Sigma\text{: diagonal singular values} \\ V^T\text{: right singular vectors } (n \times n) \end{aligned}$$
  • Truncated SVD: keep top-k → compression / PCA
  • Recommender systems: matrix factorization via SVD
  • LSA: SVD on TF-IDF matrix for text

Common Mistakes & Misconceptions
  • Matrix multiplication order confusion — AB ≠ BA in general (matrix multiplication is not commutative). A common bug: applying a weight matrix W to a batch of vectors where the dimensions are transposed. Always track shapes explicitly: (batch × d_in) @ (d_in × d_out) = (batch × d_out)
  • Confusing eigenvalues and singular values — eigendecomposition only applies to square matrices; SVD applies to any matrix. PCA uses eigendecomposition of the covariance matrix (square) OR equivalently SVD of the data matrix (any shape). SVD is numerically more stable for PCA on wide feature matrices
  • "Rank tells you nothing useful" — a rank-deficient (X^T X) matrix means you can't invert it for OLS — the system is underdetermined (more features than independent equations). In practice, multicollinear features produce near-zero eigenvalues in the covariance matrix, signalling rank deficiency even before full rank failure
  • Ignoring broadcasting rules in numpy/PyTorch — tensor operations broadcast dimensions from right-to-left. A (3, 1) tensor added to a (1, 4) tensor produces (3, 4), not an error. Misunderstanding broadcasting causes silent shape bugs — intermediate results have unexpected dimensions
How It's Used at Top Companies
  • Google (TPU matrix units): TPU hardware is fundamentally a matrix multiply unit (MXU) — the entire chip is optimised for large dense matrix multiplications; transformer training is essentially a sequence of matrix multiplies that maps perfectly to TPU architecture
  • Recommender systems (matrix factorisation): user-item interaction matrix factorised via SVD or ALS into user and item embedding matrices; the dot product of user and item embeddings approximates the original rating — SVD is the mathematical core of collaborative filtering
  • Computer graphics / robotics: 3D transformations (rotation, scaling, translation) are matrix operations; rotation matrices form a group (SO(3)); deep learning for 3D (PointNet, NeRF) heavily uses these linear algebra structures
  • Numerical stability in LLMs: attention logits scale with d_k; scaling by 1/√d_k is a direct application of variance analysis — without it, large d_k causes dot products with high variance, pushing softmax to saturation (near-zero gradients)
🎯 Interview Gold

What to say: "Linear algebra underpins virtually everything in ML. Neural network forward passes are sequences of matrix multiplications. Attention is scaled dot-product (QK^T/√d_k) between query and key matrices. PCA eigendecomposes the covariance matrix to find orthogonal directions of maximum variance. SVD factorises any matrix — the mathematical core of collaborative filtering and LSA. The most common debugging skill: tracking tensor shapes — if your gradient has the wrong shape, you made a linear algebra error."

Common mistake to avoid saying: "AB = BA." — Matrix multiplication is generally not commutative. The order of operations in a neural network forward pass matters; transposing weight matrices at the wrong point produces a shape that looks correct but computes something entirely different.

Expect the follow-up: "Why is SVD preferred over eigendecomposition for PCA in practice?" — Answer: eigendecomposition of X^TX requires explicitly computing the covariance matrix (numerically expensive and can amplify floating-point errors for wide matrices). SVD of X directly gives the principal components without forming X^TX — more numerically stable, especially when n < d (more features than samples). scipy.linalg.svd with full_matrices=False is the standard implementation.
