Attention & Transformer Architecture
The complete architectural lineage from vanilla RNNs to GPT — understanding exactly why each component was invented, what problem it solved, and how they compose into the architecture that powers modern AI.
Part 1 — The Problem: Sequence Modelling Before Attention
Vanilla RNN — The Goldfish
At each timestep: hₜ = tanh(Wₓxₜ + Wₕhₜ₋₁ + b). The hidden state must encode everything seen so far in a fixed-size vector. Works for short sequences. Fails on long ones — gradients vanish over 10+ steps, early tokens are forgotten.
  • Vanishing gradient: multiplying by Wₕ at every step → gradients shrink to zero
  • Sequential: each step depends on the previous — cannot parallelise training
  • Fixed memory: entire history must fit in hₜ — information bottleneck
LSTM — Adding Controlled Memory
Introduced a cell state Cₜ — a "conveyor belt" running alongside hₜ. Three gates (forget, input, output) learn to selectively erase, write, and read from Cₜ, allowing gradients to flow across hundreds of timesteps.
  • Fixes: vanishing gradients, long-range dependencies
  • Still sequential: cannot parallelise — slow to train on long documents
  • Still bottlenecked: must compress source into one vector for translation
The Seq2Seq Bottleneck (Sutskever et al., 2014)
For machine translation, two LSTMs were chained: an encoder read the full source, producing a single context vector c = h_T, and a decoder generated the target from it. This worked — but a single vector had to summarise an entire sentence.
$$\begin{aligned} \text{Encoder: } h_t &= \text{LSTM}(x_t, h_{t-1}) \quad\rightarrow\quad c = h_T \text{ (final hidden state only)} \\ \text{Decoder: } s_t &= \text{LSTM}(y_{t-1},\, s_{t-1},\, c),\quad y_t = \text{softmax}(W s_t) \\ &\text{Problem: entire source sentence} \rightarrow \text{one vector } c \rightarrow \text{bottleneck} \end{aligned}$$
⚠️ The Bottleneck Problem

Translating "The cat sat on the mat because it was tired" requires knowing that "it" refers to "the cat" — but by the time the decoder generates "it", that information has been diluted through many LSTM steps into the single vector c. Quality degrades sharply for sentences longer than ~20 words.

Part 2 — The Solution: Attention Mechanism (Bahdanau et al., 2015)
The Core Idea — Dynamic Context
Instead of compressing the source into one vector, keep all encoder hidden states and let the decoder attend to whichever ones are relevant at each step. The attention weight αₜᵢ says: "when generating output word t, how much should I look at source position i?"
$$\begin{aligned} e_{ti} &= \text{score}(s_{t-1}, h_i) \quad \text{(alignment score — how relevant is } h_i \text{ to decoder state } s_{t-1}\text{?)} \\ \alpha_{ti} &= \frac{\exp(e_{ti})}{\sum_j \exp(e_{tj})} \quad \text{(softmax over all encoder positions)} \\ c_t &= \sum_i \alpha_{ti}\, h_i \quad \text{(weighted sum — dynamic context vector per decoder step)} \end{aligned}$$
Encoder-Decoder with Attention — decoder attends to all encoder states, not just the last
Part 3 — Scaled Dot-Product & Multi-Head Attention
Scaled Dot-Product Attention — The Core Operation
The Transformer generalises attention into a fully learnable operation. Each token is projected into three roles simultaneously: a Query (what am I looking for?), a Key (what do I advertise?), and a Value (what do I output when matched?). Attention scores are computed as dot-products between all Q–K pairs, scaled and softmaxed to produce weights over V.
$$\begin{aligned} \text{Attention}(Q,K,V) &= \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)\!V \\[6pt] Q = XW^Q &\quad \text{[queries: "what am I looking for?"]} \\ K = XW^K &\quad \text{[keys: "what information do I have?"]} \\ V = XW^V &\quad \text{[values: "what to output if matched?"]} \\ d_k &= \text{key/query dimension (scale factor prevents softmax saturation)} \end{aligned}$$
Why √dₖ Scaling?
Dot products grow as O(√dₖ) with dimension — without scaling, large dₖ produces very large logits → softmax saturates → near-zero gradients for all non-maximum positions. Dividing by √dₖ keeps the softmax in a gradient-friendly range.
💡 Rule of thumb

If dₖ = 64 (typical), divide by 8. If dₖ = 512 (large model), divide by ~22. The scale factor is always √dₖ regardless of model size.

Self-Attention vs Cross-Attention
  • Self-Attention: Q, K, V all from the same sequence. Each token attends to all other tokens in the same sequence. Used in both Encoder and Decoder self-attention blocks.
  • Cross-Attention: Q from decoder, K and V from encoder. Decoder positions attend to relevant encoder positions — the "lookup" that connects encoder knowledge to decoder generation.
  • Masked Self-Attention: causal mask prevents each position from attending to future tokens. Used in decoder-only models (GPT) at training time.
Multi-Head Attention — Learning Multiple Perspectives in Parallel
Running a single attention head learns one relationship pattern. Running h heads in parallel, each with its own Q/K/V projections, lets the model simultaneously attend to syntax, semantics, coreference, and positional patterns — then concatenate and project the results.
$$\begin{aligned} \text{head}_i &= \text{Attention}(QW_i^Q,\; KW_i^K,\; VW_i^V) \\ \text{MultiHead}(Q,K,V) &= \text{Concat}(\text{head}_1,\ldots,\text{head}_h)\,W^O \\ &\text{Typical: } h=8 \text{ heads},\; d_k = d_v = d_{\text{model}}/h = 64 \end{aligned}$$
Attention: token "bank" attends — high weight to "deposit" and "$", low to other tokens
Part 4 — The Transformer Architecture ("Attention Is All You Need", 2017)
Why Replace RNNs Entirely?
Attention alone already allows any token to directly attend to any other token — O(1) path length between any two positions regardless of distance. RNNs have O(n) path length. The key insight of Vaswani et al. (2017): if attention already handles long-range dependencies, the recurrence is just adding sequential computation overhead. Remove it entirely.
  • No recurrence: all positions processed in parallel → 10–100× faster training on GPUs
  • O(1) path length: any token can attend to any other in a single layer
  • Position-invariant: must inject positional information explicitly via PE
  • O(n²) memory: QKᵀ matrix is seq_len × seq_len — scales poorly to very long sequences
Positional Encoding — Injecting Order Without Recurrence
Self-attention is permutation-invariant — it produces the same output regardless of token order. Positional encodings are added to the token embeddings before the first layer to inject position information. The original paper used sinusoidal encodings; modern LLMs use learned or rotary (RoPE) encodings.
$$\begin{aligned} PE(\text{pos}, 2i) &= \sin\!\left(\frac{\text{pos}}{10000^{2i/d_{\text{model}}}}\right) \\ PE(\text{pos}, 2i+1) &= \cos\!\left(\frac{\text{pos}}{10000^{2i/d_{\text{model}}}}\right) \\ \text{Input to each layer} &= \text{token embedding} + \text{positional encoding} \end{aligned}$$
Different frequencies encode different scales: high-frequency dimensions change rapidly between adjacent positions (local info), low-frequency dimensions change slowly across the sequence (global position). RoPE (used in LLaMA, GPT-NeoX) rotates Q and K vectors instead of adding to embeddings — better extrapolation to unseen sequence lengths.
Transformer Encoder Block — Complete Forward Pass
One Transformer encoder layer = two sub-layers (self-attention + FFN), each wrapped with a residual connection and Layer Norm. BERT uses 12 such layers; GPT-3 uses 96.
$$\begin{aligned} &\text{Input: } x \in \mathbb{R}^{\text{seq\_len} \times d_{\text{model}}} \\[4pt] &\textbf{Sub-layer 1 — Multi-Head Self-Attention:} \\ &x' = \text{LayerNorm}(x) \quad\text{[pre-norm variant]} \\ &\text{att} = \text{MultiHeadSelfAttention}(x', x', x') \\ &x = x + \text{att} \quad\text{(residual connection)} \\[4pt] &\textbf{Sub-layer 2 — Feed-Forward Network:} \\ &x' = \text{LayerNorm}(x) \\ &\text{ffn} = \text{Linear}_2(\text{GELU}(\text{Linear}_1(x'))) \quad\text{[}d \to 4d \to d\text{]} \\ &x = x + \text{ffn} \quad\text{(residual connection)} \\[4pt] &\text{Repeat } N \text{ times (}N{=}12 \text{ for BERT-base, } N{=}96 \text{ for GPT-3)} \end{aligned}$$
  • Residual connections: enable gradient flow through 100+ layers without vanishing
  • Layer Norm (not Batch Norm): position-independent, handles variable sequence lengths
  • FFN = knowledge memory: 2/3 of parameters live here — stores factual associations
  • 4× expansion: dₘₒdₑₗ → 4dₘₒdₑₗ → dₘₒdₑₗ; GELU activates the "memory" selectively

Modern Architecture Improvements
  • RoPE (Rotary Position Embedding): rotates Q and K vectors by position angle — better length generalisation than sinusoidal. Used in LLaMA, GPT-NeoX.
  • GQA (Grouped Query Attention): fewer K/V heads than Q heads — reduces KV cache memory by 4–8× at inference with minimal quality loss. Used in Mistral, LLaMA-3.
  • SwiGLU activation: replaces ReLU in FFN with x · σ(W₁x) ⊗ W₂x — better gradients. Used in LLaMA, PaLM.
  • RMSNorm: simpler than LayerNorm — omits the mean-centring step. Faster and equally effective.
  • Flash Attention: tiling trick — computes attention in SRAM instead of HBM, reducing memory from O(n²) to O(n) while maintaining exact results. 4× faster.

📖 The Library Analogy

An RNN reads a book page by page, maintaining a single notepad summary. By chapter 20 it has mostly forgotten chapter 1. An LSTM has a better notepad with sticky notes — but still reads page by page. A Transformer gets the entire book at once, draws lines between every relevant passage simultaneously (attention), and builds its understanding in parallel across all pages. The FFN is the interpreter that extracts facts from the underlined passages. That's why Transformers understand "it" in chapter 20 refers to the cat mentioned in chapter 1 — both are visible at once.

🎯 Interview Gold

What to say: "Attention lets every position directly attend to every other — replacing sequential recurrence with parallel computation. Three design choices make it work: residual connections for gradient flow through depth; Layer Norm (not BatchNorm) for sequence-length-independent training stability; multi-head attention so the model simultaneously tracks syntax, semantics, and coreference with independent subspaces. The FFN layers between attention blocks act as key-value memory — factual knowledge is stored in FFN weights, which is why wider models know more facts. Modern decoder-only GPT-style models drop the cross-attention encoder and train autoregressively — simpler and generalist enough to handle any task via prompting."

Common mistake to avoid saying: "The √d_k scaling is just a trick." — It has a principled reason: without scaling, dot products grow with dimensionality (variance scales with d_k), pushing softmax into saturation regions where gradients vanish. Dividing by √d_k keeps the dot product variance at ~1 regardless of d_k.

Expect the follow-up: "What is GQA (Grouped Query Attention) and why is it used?" — Answer: standard MHA has a separate K and V head per Q head. GQA groups multiple Q heads to share one K/V head. Result: KV-cache is much smaller (critical for long-context inference), with minimal accuracy loss. LLaMA 2/3, Mistral, and most production LLMs use GQA — the KV-cache is the primary memory bottleneck at inference, so reducing its size is a priority.

Common Mistakes & Misconceptions
  • O(n²) is always a problem — at typical inference context lengths (512–8K tokens), O(n²) is fast. The bottleneck is usually the FFN layers (O(n·d_model·d_ff)) not the attention matrix. FlashAttention's value is primarily memory (avoids materialising the n×n matrix) not raw FLOPs
  • Confusing encoder and decoder attention masking — encoder self-attention is bidirectional (each token attends to all others); decoder self-attention is causal (each token only attends to past tokens via a triangular mask). Mixing these up is a common implementation bug
  • "More heads = better" — attention heads with very small d_k (=d_model/n_heads) have limited expressivity per head. Standard practice: d_k = 64 (d_model=1024, 16 heads). Fewer but wider heads can outperform many tiny heads
  • Not caching KV during autoregressive inference — without a KV cache, each new token requires recomputing K and V for the entire previous context: O(n²) per token. The KV cache stores past K/V, reducing each decode step to O(n) — critical for production serving latency
How It's Used at Top Companies
  • All major LLM providers (OpenAI, Anthropic, Google): FlashAttention-2/3 is universal in training and inference — IO-aware attention avoids reading/writing the full n×n attention matrix to HBM, reducing memory bandwidth 3–10× and enabling longer contexts
  • Meta (LLaMA 3): GQA with 8 KV heads for 70 heads of Q — KV cache is 8× smaller than MHA, enabling 128K context window serving on 80GB A100s that would otherwise OOM
  • Google (Gemini / PaLM): Multi-query attention (MQA, extreme GQA with 1 KV head) in early versions for maximum inference throughput; GQA preferred in newer models for quality/efficiency balance
  • Mistral AI: Sliding window attention (SWA) — each token attends only to a local window of W tokens; global attention via stacked layers. Reduces attention from O(n²) to O(n·W) with good long-context quality through layer stacking
