Statistics & Inference
Where probability meets data. The toolkit for making confident, honest claims from noisy observations — the foundation of A/B testing and scientific ML.
p-value — the most misunderstood number in science
P(observing data this extreme | H₀ is true). NOT the probability that H₀ is true.
⚠️ Three Things p-value Is NOT

1) Not the probability H₀ is true. 2) Not the probability your result happened by chance. 3) Not a measure of effect size. With 10M users, p < 0.001 for a 0.0001% CTR change — statistically significant, practically irrelevant. Always pair with effect size + CI.

📖 The Coin Story

You flip a coin 10 times, get 9 heads. p ≈ 1% = "if this were a fair coin, I'd see this by luck only 1% of the time." Since 1% < 5% (your threshold), you reject "it's a fair coin." p = 30%? Meh — totally plausible by chance, don't reject.

Type I Error (α) — False Positive
Reject H₀ when it's true. See an effect that doesn't exist.
🔥 Fire alarm, no fire

You ran outside for nothing. α = 0.05 means 5% chance of this mistake per test.

Type II Error (β) — False Negative
Fail to reject H₀ when it's false. Miss a real effect.
🏠 Real fire, no alarm

Power = 1 − β = probability of detecting a real effect. More data = higher power.

Confidence Interval & Hypothesis Testing
$$\begin{aligned} 95\%\ \text{CI:} \quad \bar{x} \pm 1.96 \times \frac{\sigma}{\sqrt{n}} \end{aligned}$$
5-step process: 1. H₀ (no effect) vs H₁ (effect exists) 2. Choose α (usually 0.05) 3. Collect data, pick test 4. Compute p-value 5. p < α → reject H₀ (≠ accept H₁!)
💡 CI Interpretation

A 95% CI does NOT mean "95% probability the true value is in this range." It means: if we repeated this experiment 100 times, ~95 of those intervals would contain the true value. The interval either contains the true value or it doesn't.

Common Mistakes & Misconceptions
  • Peeking at results before the experiment ends — checking significance mid-experiment and stopping early when p < 0.05 inflates Type I error severely. With α=0.05 and 5 peeks, the true false positive rate approaches 19%. Pre-register sample size via power analysis; commit to it before any data is collected
  • "p < 0.05 means the effect is real and important" — statistical significance ≠ practical significance. At n=1,000,000, a 0.001% conversion rate lift will be highly significant (p < 0.001) but economically meaningless. Always report Cohen's d or relative lift alongside p-values
  • Multiple comparisons without correction — testing 20 metrics at α=0.05 gives an expected 1 false positive just by chance. Apply Bonferroni correction (α/n tests) or Benjamini-Hochberg (controls false discovery rate) when testing multiple hypotheses simultaneously. This is especially critical in A/B tests monitoring many metrics
  • Assuming normality without checking — t-tests assume normally distributed residuals (or large n by CLT). Revenue, session time, and purchase counts are typically right-skewed. For n < 30 with skewed data, use bootstrapped confidence intervals or Mann-Whitney U instead of a t-test
  • Confusing correlation and causation in observational data — Pearson/Spearman correlation measures association; it says nothing about direction or causality. Ice cream sales and drowning rates are correlated (both increase in summer). Causal inference requires randomisation (RCT) or quasi-experimental methods (DiD, IV, RDD)
How It's Used at Top Companies
  • Airbnb / Booking.com (A/B testing): sequential testing (always-valid p-values using e-values or mixture sequential probability ratio tests) — allows peeking without inflating Type I error; replaces fixed-horizon tests for faster iteration. Power analysis determines minimum detectable effect (MDE) before each experiment
  • Netflix (experimentation platform): delta method for ratio metrics (e.g., revenue per user when both numerator and denominator vary); bootstrapped CIs for non-normal metrics (session time, engagement rates); CUPED (controlled-experiment using pre-experiment data) to reduce variance and increase power without more traffic
  • Google (Statsig / experiment infrastructure): ANOVA and regression-adjusted analysis for multivariate experiments; Bonferroni correction as default when testing multiple metrics simultaneously; holdout groups maintained permanently to measure cumulative experiment effects
  • Stripe (metric selection): pre-specified primary metric with pre-registered sample size; secondary metrics monitored with FDR control; guardrail metrics (latency, error rate) that must not degrade — the full statistical framework defined before any data collection begins
🎯 Interview Gold

What to say: "I always pair p-values with effect sizes and confidence intervals — statistical significance without practical significance is noise. For A/B tests: power analysis to pre-register sample size before peeking (peeking inflates Type I error); at massive scale, everything is 'significant' — the real question is whether the lift is worth the cost. For non-normal metrics (revenue, session time), I prefer bootstrapped CIs or Mann-Whitney U over t-tests. Multiple comparisons: Bonferroni for few tests, Benjamini-Hochberg for many."

Common mistake to avoid saying: "The A/B test was significant at p=0.03 so we shipped." — Without pre-registration, sample size calculation, and multiple comparison correction, p=0.03 may be a false positive. Interviewers at product companies will probe whether you understand the full rigour of experiment design, not just significance testing.

Expect the follow-up: "How do you determine sample size for an A/B test?" — Answer: power analysis requires four inputs: baseline metric value, minimum detectable effect (MDE — the smallest lift worth detecting), significance level α (usually 0.05), and desired power 1−β (usually 0.80). n ≈ 2 × (z_α/2 + z_β)² × σ² / δ² where δ is the MDE and σ² is metric variance. Tools: statsmodels.stats.power, or the formula directly. Always compute this before starting the experiment, not after.
