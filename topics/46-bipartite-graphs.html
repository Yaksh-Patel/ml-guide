Bipartite Graphs in ML
A fundamental graph structure where nodes split into two disjoint sets with edges only between sets — the natural model for recommendations, matching, and collaborative filtering.
What Is a Bipartite Graph?
A graph G = (U ∪ V, E) where U and V are disjoint node sets, and every edge connects a node in U to a node in V. No edges within U or within V.
$$\begin{aligned} G = (U, V, E) \\ U = {users}, V = {items/movies/products} \\ E = {(u,v) | user u interacted with item v} \\ Bipartite iff: no odd-length cycles (can 2-color the graph) \end{aligned}$$
Bipartite graph: users ↔ items — edges = interactions (ratings, purchases, clicks)
Bipartite Graph → Adjacency / Rating Matrix
$$\begin{aligned} R[u,v] &= \text{rating user } u \text{ gave item } v \;(0 \text{ if no interaction}) \\ &\text{sparse matrix, shape: } |U| \times |V| \\ R &\approx U \cdot V^T \quad U \in \mathbb{R}^{|U|\times k},\; V \in \mathbb{R}^{|V|\times k} \\ \hat{R}[u,v] &= U[u,:]\cdot V[v,:]^T \\ \text{Minimize: } & \sum_{(u,v)\in\text{observed}} (R[u,v] - U[u,:]\cdot V[v,:]^T)^2 + \lambda\|U\|^2 + \lambda\|V\|^2 \end{aligned}$$
Collaborative Filtering via Bipartite Graph
  • User-based CF: find similar users (similar rows in R), recommend what they liked
  • Item-based CF: find similar items (similar columns in R), recommend similar to what user liked
  • Matrix factorization (SVD): latent factors capture hidden user preferences and item characteristics
  • Cold start problem: new user or item has no edges in bipartite graph → can't recommend
Graph Neural Networks on Bipartite Graphs
  • LightGCN: propagate embeddings along bipartite edges — user embedding influenced by connected items, item embedding influenced by connected users
  • PinSage: Pinterest's GNN on bipartite pin-board graph — samples neighborhoods, aggregates features
  • Bipartite matching: assignment problem — maximum matching (e.g., Hopcroft-Karp algorithm) for allocation, ridesharing, job assignment

🗃️ SQL
# Check if graph is bipartite (BFS 2-coloring)
from collections import deque
def is_bipartite(graph, n):
    color = [-1] * n
    for start in range(n):
        if color[start] != -1: continue
        color[start] = 0
        q = deque([start])
        while q:
            u = q.popleft()
            for v in graph[u]:
                if color[v] == -1:
                    color[v] = 1 - color[u]
                    q.append(v)
                elif color[v] == color[u]:
                    return False   # odd cycle → not bipartite
    return True
🎯 Interview Gold

"In recommender systems, the bipartite user-item graph IS the data. Matrix factorization decomposes it into user and item latent embeddings — the dot product of a user's embedding and item's embedding predicts the rating. Modern GNN-based recommenders (LightGCN, NGCF) message-pass along this bipartite graph: a user's embedding is updated by the embeddings of items they interacted with, and vice versa — iteratively enriching representations. The cold-start problem is a graph problem: a new node has no edges, so its embedding is uninitialized."
