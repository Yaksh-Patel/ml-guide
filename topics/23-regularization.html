Regularization
The surgeon that trims model complexity — adds a penalty to the loss function to prevent overfitting and improve generalization.
$$\text{Total Loss} = \underbrace{\text{Data Loss}}_{\text{fit}} + \lambda \times \underbrace{\text{Regularization Penalty}}_{\text{complexity}}$$
L1 Regularization — Lasso
$$\text{Penalty} = \lambda \sum_i |w_i|$$
  • Pushes coefficients to exactly zero — automatic feature selection
  • Produces sparse models — most features get wᵢ = 0
  • Non-differentiable at 0 — requires subgradient methods
  • MAP interpretation: Laplace prior on weights
Feature selectionSparse models
L2 Regularization — Ridge
$$\text{Penalty} = \lambda \sum_i w_i^2$$
  • Shrinks all coefficients proportionally toward zero — never exactly 0
  • Smooth penalty — differentiable everywhere
  • Better when all features are somewhat relevant
  • Handles multicollinearity well (distributes weight)
  • MAP interpretation: Gaussian prior on weights
ShrinkageStable
Dropout — Regularization for Neural Networks
Randomly zero out each neuron with probability p during training (typically p=0.5 for hidden, p=0.2 for input). Forces the network to learn redundant representations — no single neuron can dominate.
$$\begin{aligned} z_i &= \text{activation} \times \frac{\text{Bernoulli}(1-p)}{1-p} \quad \text{(inverted dropout)} \\ &\quad \text{At inference: use all neurons (no scaling needed)} \end{aligned}$$
  • Ensemble effect: trains ~2^n sub-networks implicitly
  • Apply after activation; before input to next layer
  • p=0.5 for fully connected; p=0.1–0.2 for convolutional; not needed with BatchNorm

📖 The Strict Editor

Regularization is an editor charging you for every word: L1 cuts entire sentences (zeros features). L2 makes every sentence shorter. Dropout randomly silences 50% of your team during rehearsal — they learn to compensate for absent colleagues, making each individual more capable.

Common Mistakes & Misconceptions
  • Confusing L1 and L2 effects on sparsity — L1 (Lasso) creates sparse solutions because its gradient is constant (±1), so small weights get driven exactly to zero. L2 (Ridge) has a gradient proportional to the weight, so it shrinks weights but rarely zeros them out
  • Applying the same λ to all features without scaling — regularization penalizes large weights, but "large" depends on feature scale. Always StandardScale features before applying L1/L2; otherwise high-scale features are penalized more than low-scale features of equal importance
  • "Dropout replaces all other regularization" — dropout is effective for fully connected layers but insufficient alone. Modern best practice: dropout + weight decay (AdamW) + batch normalization + data augmentation — layers of complementary regularization
  • Choosing λ by intuition — regularization strength is a critical hyperparameter. Always tune via cross-validation over a log-scale grid (e.g., [0.0001, 0.001, 0.01, 0.1, 1, 10]); intuition about the right λ is consistently wrong
  • Using L1 when features are correlated — among a group of correlated features, L1 arbitrarily picks one and zeros the rest. If all correlated features are meaningful, use L2 or Elastic Net (L1+L2) instead
How It's Used at Top Companies
  • Google (neural machine translation): dropout (rate=0.1–0.3) on all transformer layers + weight decay; label smoothing as an additional regularizer that prevents overconfident softmax outputs — now standard in all transformer training
  • Netflix / Spotify (collaborative filtering): L2 regularization on user and item embedding matrices prevents extreme embedding magnitudes for users/items with few interactions (cold start regularization)
  • Genomics / bioinformatics: Elastic Net regression for gene expression prediction — L1 for feature selection (most of 20,000 genes are irrelevant), L2 for stability among correlated gene groups
  • Credit scoring: L2 (Ridge) regularization on logistic regression scorecard prevents individual feature coefficients from becoming extreme; regulatory requirement that no single feature can dominate the prediction
🎯 Interview Gold

What to say: "Regularization encodes the prior belief that simpler models generalise better. L1 (Lasso) for feature selection — penalizes with constant gradient, drives small weights to exactly zero, producing sparse solutions. L2 (Ridge) when most features matter — shrinks proportionally, stable with correlated features. For deep learning: dropout + weight decay (AdamW) + data augmentation + early stopping — each addresses a different failure mode. λ is always tuned via CV over a log-scale grid; never guessed."

Common mistake to avoid saying: "I set dropout=0.5 based on the original paper." — 0.5 was optimal for large fully-connected layers in 2012 AlexNet. Modern transformer layers typically use 0.1. Always tune dropout rate; it's dataset and architecture dependent.

Expect the follow-up: "Why does L1 produce sparse solutions but L2 doesn't?" — Answer: the L1 penalty has a constant subgradient of ±1 regardless of weight magnitude. Even a tiny weight gets pushed by the full ±λ gradient — it reaches zero and stays there (no gradient to push it off zero). L2's gradient is proportional to the weight: as weight → 0, the gradient → 0, so the update force vanishes before reaching zero. L1 has a "sharp corner" at zero in the penalty geometry; L2 has a smooth bowl.
