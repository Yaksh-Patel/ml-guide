Regularization
The surgeon that trims model complexity — adds a penalty to the loss function to prevent overfitting and improve generalization.
$$\text{Total Loss} = \underbrace{\text{Data Loss}}_{\text{fit}} + \lambda \times \underbrace{\text{Regularization Penalty}}_{\text{complexity}}$$
L1 Regularization — Lasso
$$\text{Penalty} = \lambda \sum_i |w_i|$$
  • Pushes coefficients to exactly zero — automatic feature selection
  • Produces sparse models — most features get wᵢ = 0
  • Non-differentiable at 0 — requires subgradient methods
  • MAP interpretation: Laplace prior on weights
Feature selectionSparse models
L2 Regularization — Ridge
$$\text{Penalty} = \lambda \sum_i w_i^2$$
  • Shrinks all coefficients proportionally toward zero — never exactly 0
  • Smooth penalty — differentiable everywhere
  • Better when all features are somewhat relevant
  • Handles multicollinearity well (distributes weight)
  • MAP interpretation: Gaussian prior on weights
ShrinkageStable
Dropout — Regularization for Neural Networks
Randomly zero out each neuron with probability p during training (typically p=0.5 for hidden, p=0.2 for input). Forces the network to learn redundant representations — no single neuron can dominate.
$$\begin{aligned} z_i &= \text{activation} \times \frac{\text{Bernoulli}(1-p)}{1-p} \quad \text{(inverted dropout)} \\ &\quad \text{At inference: use all neurons (no scaling needed)} \end{aligned}$$
  • Ensemble effect: trains ~2^n sub-networks implicitly
  • Apply after activation; before input to next layer
  • p=0.5 for fully connected; p=0.1–0.2 for convolutional; not needed with BatchNorm

📖 The Strict Editor

Regularization is an editor charging you for every word: L1 cuts entire sentences (zeros features). L2 makes every sentence shorter. Dropout randomly silences 50% of your team during rehearsal — they learn to compensate for absent colleagues, making each individual more capable.

🎯 Interview Gold

"I think of regularization as a prior belief that simpler models generalize better. L1 when I suspect many features are irrelevant (feature selection baked in). L2 when most features matter but I want to prevent extreme weights. For deep learning: dropout + weight decay (AdamW) + early stopping + data augmentation. The strength λ is always tuned via cross-validation — too much and you underfit, too little and you overfit."
