Cross-Validation
The honest way to estimate how your model will perform on unseen data — without wasting data on a fixed test set during development.
k-Fold Cross-Validation
Split data into k equal folds. Train on k−1 folds, validate on the held-out fold. Rotate k times. Average the k validation scores.
5-Fold Cross-Validation — each fold takes a turn as validation
CV Variants
TypeWhen to Use
k-Fold (k=5 or 10)Default for tabular data
Stratified k-FoldImbalanced classes — preserves class ratio in each fold
Leave-One-Out (LOO)Tiny datasets (<50 rows); expensive
TimeSeriesSplitTime series — never use future data to validate past
Group k-FoldGroups must stay intact (patient IDs, user sessions)
Nested CVHyperparameter tuning — outer loop for eval, inner for tuning
Common CV Mistakes
  • Data leakage from preprocessing: fit scaler on all data before splitting → test info leaks into training
  • Peeking at test set: using test performance to make modeling decisions inflates estimates
  • No stratification: imbalanced classes can produce folds with 0 positives
  • Time series with random split: future data leaks into training — always use temporal split
⚠️ The Golden Rule

Preprocessing (scaling, imputation, encoding) must be fit ONLY on the training fold, then applied to validation. Use sklearn Pipelines to enforce this automatically. Never fit on the full dataset before splitting.

🎯 Interview Gold

"I always use stratified k-fold (k=5) for classification with imbalanced classes, and TimeSeriesSplit for temporal data. I wrap all preprocessing in a Pipeline so feature scaling is re-fit on each fold's training data — this prevents leakage. Nested CV when I need to tune hyperparameters and estimate generalization performance simultaneously — but it's expensive."
