Calibration & Explainability
A model that's 70% confident should be right 70% of the time. Calibration checks this. Explainability turns black boxes into stories stakeholders trust.
Model Calibration
A well-calibrated model's predicted probabilities match observed frequencies.
  • Reliability diagram: x-axis = predicted prob buckets, y-axis = actual frequency. Perfect calibration = diagonal.
  • Overconfident: sigmoid curve below diagonal (predictions too extreme)
  • Underconfident: S-curve above diagonal (predictions too conservative)
  • ECE (Expected Calibration Error): weighted average of |pred − actual| per bucket
Fixing calibration:
  • Platt scaling: fit logistic regression on validation scores → calibrated prob
  • Isotonic regression: non-parametric, more flexible, needs more data
  • Temperature scaling: divide logits by T before softmax (for NNs)
SHAP — SHapley Additive exPlanations
Allocates the model's prediction to each feature based on Shapley values from cooperative game theory.
$$\begin{aligned} \phi_i = \sum_{S \subseteq F \setminus \{i\}} \frac{|S|!\,(n-|S|-1)!}{n!} \left[f(S \cup \{i\}) - f(S)\right] \\[4pt] \text{(average marginal contribution across all feature orderings)} \end{aligned}$$
  • Global SHAP: mean |SHAP| per feature = feature importance ranking
  • Local SHAP: why did THIS prediction get this score?
  • SHAP dependence plots: how does feature X affect predictions across its range?
  • TreeSHAP: O(TLD) exact computation for tree models — very fast
LIME — Local Interpretable Model-Agnostic Explanations
Fit a simple linear model locally around a single prediction to explain it. Perturb the input, see how predictions change, fit interpretable model to those changes.
  • Works for any black-box model (text, image, tabular)
  • Approximation is local — explanation only valid near the explained instance
  • Less stable than SHAP — can give different explanations for similar instances

🎯 Interview Gold

"Calibration matters especially in credit risk and medical applications where the predicted probability IS the output (e.g., P(default) determines the interest rate). XGBoost is poorly calibrated out-of-the-box — I always apply Platt scaling or isotonic regression post-hoc. For explainability, SHAP is my default — it's the only method with theoretical guarantees (efficiency, dummy, symmetry, linearity properties). I use TreeSHAP for tree models (fast and exact) and SHAP KernelExplainer for everything else."
