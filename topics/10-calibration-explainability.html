Calibration & Explainability
A model that's 70% confident should be right 70% of the time. Calibration checks this. Explainability turns black boxes into stories stakeholders trust.
Model Calibration
A well-calibrated model's predicted probabilities match observed frequencies.
  • Reliability diagram: x-axis = predicted prob buckets, y-axis = actual frequency. Perfect calibration = diagonal.
  • Overconfident: sigmoid curve below diagonal (predictions too extreme)
  • Underconfident: S-curve above diagonal (predictions too conservative)
  • ECE (Expected Calibration Error): weighted average of |pred − actual| per bucket
Fixing calibration:
  • Platt scaling: fit logistic regression on validation scores → calibrated prob
  • Isotonic regression: non-parametric, more flexible, needs more data
  • Temperature scaling: divide logits by T before softmax (for NNs)
SHAP — SHapley Additive exPlanations
Allocates the model's prediction to each feature based on Shapley values from cooperative game theory.
$$\begin{aligned} \phi_i = \sum_{S \subseteq F \setminus \{i\}} \frac{|S|!\,(n-|S|-1)!}{n!} \left[f(S \cup \{i\}) - f(S)\right] \\[4pt] \text{(average marginal contribution across all feature orderings)} \end{aligned}$$
  • Global SHAP: mean |SHAP| per feature = feature importance ranking
  • Local SHAP: why did THIS prediction get this score?
  • SHAP dependence plots: how does feature X affect predictions across its range?
  • TreeSHAP: O(TLD) exact computation for tree models — very fast
LIME — Local Interpretable Model-Agnostic Explanations
Fit a simple linear model locally around a single prediction to explain it. Perturb the input, see how predictions change, fit interpretable model to those changes.
  • Works for any black-box model (text, image, tabular)
  • Approximation is local — explanation only valid near the explained instance
  • Less stable than SHAP — can give different explanations for similar instances

Common Mistakes & Misconceptions
  • Assuming tree models are well-calibrated — Random Forests and XGBoost tend to produce probabilities pushed toward 0 and 1 (overconfident). Always check a reliability diagram; apply CalibratedClassifierCV with isotonic regression post-hoc
  • Using LIME for global explanations — LIME is a local explainer; each explanation is only valid near one prediction. For global feature importance, use SHAP's mean absolute SHAP value summary. Using LIME globally misleads
  • Confusing feature importance with causation — SHAP tells you which features the model relies on, not which features causally drive the outcome. A correlated proxy feature (e.g., zip code as proxy for income) can have high SHAP but removing it may not change the real-world outcome
  • Platt scaling vs isotonic regression choice — Platt scaling assumes a sigmoid relationship between scores and probabilities (works well for SVMs, naive Bayes). Isotonic regression is non-parametric and more flexible but needs more calibration data (>1000 examples). For small calibration sets, use Platt
  • Ignoring calibration drift in production — a model can maintain good AUC-ROC after concept drift while its calibration degrades severely. Monitor Expected Calibration Error (ECE) in production alongside discrimination metrics
How It's Used at Top Companies
  • Lending Club / credit firms: predicted P(default) directly determines the interest rate offered; miscalibrated probabilities translate to mispriced loans and direct financial loss — calibration is validated monthly on recent cohorts
  • Google (ads bidding): calibrated CTR predictions feed directly into the auction price calculation; a 2× miscalibration in predicted CTR causes advertisers to overbid or underbid by 2×, breaking auction theory
  • FICO (credit scores): SHAP used to generate adverse action notices — legally required explanation of why a loan was denied, formatted as "top 4 reasons" for each applicant
  • Healthcare AI (FDA-regulated): SHAP + calibration plots required in regulatory submissions; model must be demonstrated to be reliable across demographic subgroups with calibration tested separately per group
  • Airbnb (trust & safety): LIME used for debugging individual suspicious-listing predictions; SHAP used for global model monitoring — a SHAP value shift signals data or concept drift before metrics visibly degrade
🎯 Interview Gold

What to say: "Calibration matters whenever the probability itself is the output — credit risk pricing, medical diagnosis, insurance underwriting. XGBoost is poorly calibrated out-of-the-box; I apply CalibratedClassifierCV with isotonic regression post-hoc (or Platt scaling if calibration data is small). For explainability, SHAP is my default — the only method with solid game-theoretic guarantees. TreeSHAP runs in O(n·d·T) — fast enough for production. Key distinction: SHAP explains what the model relies on; it does not establish causation."

Common mistake to avoid saying: "I used LIME for global feature importance." — LIME is strictly local. Aggregate SHAP values (mean |SHAP|) for global importance. Mixing these up signals a gap in XAI understanding.

Expect the follow-up: "What is Expected Calibration Error (ECE)?" — Answer: divide predictions into bins by predicted probability. For each bin, compute |mean predicted probability − actual positive rate|. ECE is the weighted average of these gaps. A perfectly calibrated model has ECE=0: if you say 70% confident, you should be right 70% of the time.
