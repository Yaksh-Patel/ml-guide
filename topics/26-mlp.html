Perceptron & Multi-Layer Perceptron
From the simplest possible model (a weighted sum) to the universal function approximator that powers modern AI — the difference is just layers and non-linearities.
Perceptron — The Atom
$$\begin{aligned} z = w_1x_1 + w_2x_2 + ... + w_nx_n + b = w^Tx + b \\ output = activation(z) \end{aligned}$$
Learned by updating weights toward correct outputs (perceptron learning rule). Can only learn linearly separable patterns — hence the need to stack layers.
Multi-Layer Perceptron — forward pass signal flow
Forward Pass — Layer by Layer
$$\begin{aligned} h^1 = activation(W^1x + b^1) \\ h^2 = activation(W^2h^1 + b^2) \\ \hat{y} = \text{softmax}(W^3h^2 + b^3) \quad \text{(classification)} \\ \hat{y} = W^3h^2 + b^3 \quad \text{(regression)} \end{aligned}$$
Without activation functions, stacking linear layers collapses to a single linear layer: W₃(W₂(W₁x)) = Wx. The non-linearity is what enables learning non-linear decision boundaries.

Common Mistakes & Misconceptions
  • Using MLPs for small tabular datasets (<100K rows) — XGBoost and Random Forest almost always outperform MLPs on structured tabular data at this scale. Neural networks need large data to overcome their optimization difficulty. Always benchmark against tree models first
  • Not normalizing inputs — unnormalized inputs cause gradient imbalance across layers; some neurons receive huge gradients, others tiny ones. Always StandardScale or MinMaxScale inputs before any neural network
  • "More layers = better" — depth enables abstraction but creates optimization difficulties (vanishing gradients, slower training). Start shallow (2–3 hidden layers), add depth only when clearly bottlenecked. BatchNorm + residual connections enable training of very deep networks
  • Using sigmoid activations in hidden layers — sigmoid saturates (output near 0 or 1, gradient ≈ 0) causing vanishing gradients. Only use sigmoid at the output layer for binary classification. Hidden layers: ReLU or GELU
  • Training without a validation loss curve — the validation loss curve is the primary diagnostic tool for neural networks. Always plot it: diverging train/val loss = overfitting; both high = underfitting; val loss oscillating = learning rate too high
How It's Used at Top Companies
  • Google (wide & deep / DCN for recommendations): MLP component handles dense feature interactions in production recommender systems; the "deep" side of wide&deep learns combinatorial feature interactions automatically
  • Stripe (fraud — tabular neural nets): TabNet or standard MLP on engineered tabular features; competitive with XGBoost when features are continuous and dense, with the added benefit of integrated feature attention
  • Booking.com: MLPs for price and availability prediction on tabular features; carefully validated against LightGBM; neural nets adopted only where they demonstrably win on validation metrics
  • Physics / scientific ML: physics-informed neural networks (PINNs) use MLPs to approximate differential equation solutions — one of the few domains where neural nets dominate even small-data settings due to continuous function approximation needs
🎯 Interview Gold

What to say: "Universal approximation theorem: a single hidden layer with enough neurons can approximate any continuous function. But width scales poorly — depth is more efficient. Each layer learns increasingly abstract representations: pixels → edges → shapes → objects. For tabular data <100K rows, XGBoost almost always wins. MLPs shine on unstructured data (images, text) and at massive scale. Standard recipe: ReLU/GELU activations + BatchNorm + dropout + AdamW + cosine LR schedule."

Common mistake to avoid saying: "I'll use a neural network for this tabular dataset." — Without benchmarking against XGBoost first, this signals unfamiliarity with where neural nets actually win. On tabular data, always establish tree model baselines before reaching for neural nets.

Expect the follow-up: "Why does depth help more than width?" — Answer: a deep network can represent exponentially more functions with the same parameter count as a shallow wide network. Each layer composes functions learned by previous layers — depth enables hierarchical feature reuse. Width at a single layer grows the function class linearly; depth grows it exponentially.
