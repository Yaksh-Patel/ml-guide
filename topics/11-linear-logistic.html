Linear & Logistic Regression
The workhorse baseline — fast, interpretable, and the right starting point for virtually every ML project. If you can't beat logistic regression, you shouldn't ship the complex model.
Linear Regression
Predict continuous output. Minimize sum of squared residuals (OLS).
$$\begin{aligned} \hat{y} &= \beta_0 + \beta_1 x_1 + \cdots + \beta_n x_n = X\boldsymbol{\beta} \\ \hat{\boldsymbol{\beta}} &= (X^TX)^{-1}X^Ty \quad \text{(analytical solution)} \\ \mathcal{L} &= \textstyle\sum(y_i - \hat{y}_i)^2 \quad \text{(MSE)} \end{aligned}$$
Logistic Regression
Predict binary probability. Sigmoid squashes linear output to [0,1].
$$\begin{aligned} \mathbb{P}(y=1 \mid x) &= \sigma(w^Tx+b) = \dfrac{1}{1+e^{-z}} \\ \mathcal{L} &= -\bigl[y\log\hat{y} + (1-y)\log(1-\hat{y})\bigr] \quad \text{(BCE)} \\ &\quad\text{No closed form} \rightarrow \text{gradient descent} \end{aligned}$$
Assumptions — must know cold
  • Linearity: linear relationship between features and output (or log-odds for logistic)
  • Independence: observations independent — time series violates this
  • No multicollinearity: correlated features make coefficients unstable — check with VIF > 10
  • Homoscedasticity (linear): constant variance of residuals across predictions
  • Normality of residuals (linear): needed for valid coefficient CIs
  • No influential outliers: one bad point can drag the whole line

Interpretation of Coefficients
  • Linear: βᵢ = change in ŷ per unit increase in xᵢ (holding others fixed)
  • Logistic: βᵢ = change in log-odds per unit increase in xᵢ
  • Odds ratio: exp(βᵢ) — multiply odds by this per unit increase
  • Standardize features first to compare coefficient magnitudes
📖 Straight Road Analogy

Linear regression drives on a perfectly straight road. Real data has curves, hills, roundabouts. These models draw a straight line through a spiral staircase. Works brilliantly when the relationship is truly linear — but that's rarer than you think.

Common Mistakes & Misconceptions
  • Not checking assumptions before shipping — linearity, independence, homoscedasticity, no multicollinearity. Violating them doesn't crash the model but makes coefficients uninterpretable and CIs invalid. Always plot residuals vs fitted values
  • Confusing coefficient magnitude with importance — a large β doesn't mean an important feature if the feature scale is tiny. Standardize features (z-score) before comparing coefficient magnitudes
  • Multicollinearity blindspot — correlated features (e.g., height_cm and height_ft) make individual coefficients unstable and sign-flip unpredictably. Check VIF; drop or combine correlated features
  • Treating logistic regression probability as calibrated without checking — logistic regression is generally well-calibrated but not perfectly. Always check a calibration plot (reliability diagram) before using the probability output directly
  • Using accuracy to evaluate logistic regression on imbalanced data — use AUC-ROC or PR-AUC; accuracy is misleading when one class dominates
  • Forgetting the sklearn C parameter direction — in sklearn LogisticRegression, C = 1/λ (inverse regularization strength). Smaller C = more regularization. This is opposite to intuition and a frequent bug
How It's Used at Top Companies
  • Google (Ads CTR): logistic regression with billions of sparse one-hot features was the dominant CTR model for years before deep learning; still used as the calibration layer on top of complex models
  • Credit scoring industry: logistic regression with Weight of Evidence (WoE) encoding is the regulatory gold standard — coefficients are interpretable, scorecards are auditable, and monotone constraints ensure compliance
  • Netflix A/B testing: linear regression for continuous metric lift estimation in experiments; fast, interpretable, valid under the central limit theorem at Netflix's traffic scale
  • Airbnb pricing: linear regression as the baseline for price recommendation; non-linearities handled by feature engineering (log-transforms, interaction terms) rather than model complexity
  • Healthcare / clinical trials: logistic regression is the standard for binary outcome studies (disease yes/no); odds ratios from the model are directly reported in medical literature
🎯 Interview Gold

What to say: "Logistic regression is my mandatory baseline for every classification task — fast, interpretable, and well-calibrated. I validate assumptions first: check VIF for multicollinearity, plot residuals for linearity. I always standardize features before fitting so I can compare coefficient magnitudes. In sklearn, remember C = 1/λ — smaller C means stronger L2 regularization. If XGBoost doesn't beat logistic regression by at least 1% AUC on validation, I ship logistic regression — simpler and easier to maintain in production."

Common mistake to avoid saying: "The model with the highest coefficient is the most important feature." — Coefficients are scale-dependent. Always standardize first, or use permutation importance / SHAP for fair comparison.

Expect the follow-up: "Why is logistic regression called 'regression' if it's a classifier?" — Answer: it models the log-odds as a linear regression (linear combination of features), then applies the sigmoid to map to probability. The regression is on the log-odds scale; classification is a downstream thresholding decision.
