Support Vector Machines (SVM)
Find the hyperplane that maximizes the margin between classes — geometrically principled, theoretically elegant, and surprisingly powerful in high dimensions.
Core Idea — Maximum Margin Hyperplane
$$\begin{aligned} &\text{Decision boundary: } \mathbf{w}^T\mathbf{x} + b = 0 \\ &\text{Margin} = \tfrac{2}{\|\mathbf{w}\|} \rightarrow \text{minimize } \|\mathbf{w}\|^2 \\ &\text{Hard margin: } \min \tfrac{1}{2}\|\mathbf{w}\|^2 \;\text{s.t.}\; y_i(\mathbf{w}^T\mathbf{x}_i+b) \geq 1 \\ &\text{Soft margin: } \min \tfrac{1}{2}\|\mathbf{w}\|^2 + C\textstyle\sum_i\xi_i \;\text{s.t.}\; y_i(\mathbf{w}^T\mathbf{x}_i+b) \geq 1-\xi_i \end{aligned}$$
Support vectors: the training points closest to the boundary — the only ones that matter. Remove any other point and the boundary doesn't change.
SVM: maximum margin hyperplane with support vectors
The Kernel Trick
Map features to higher-dimensional space where they become linearly separable — implicitly, without explicitly computing the mapping.
$$\begin{aligned} K(\mathbf{x},\mathbf{z}) &= \varphi(\mathbf{x})^T\varphi(\mathbf{z}) \quad \text{(inner product in feature space)} \\ \text{Linear:} &\quad K = \mathbf{x}^T\mathbf{z} \\ \text{Polynomial:} &\quad K = (\mathbf{x}^T\mathbf{z} + c)^d \\ \text{RBF/Gaussian:} &\quad K = \exp(-\gamma|\mathbf{x}-\mathbf{z}|^2) \quad\text{(most common)} \\ \text{Sigmoid:} &\quad K = \tanh(\alpha\mathbf{x}^T\mathbf{z} + c) \end{aligned}$$
RBF kernel: effectively infinite-dimensional feature space. γ controls radius of influence (high γ = narrow, overfits; low γ = wide, underfits).
C Parameter (Soft Margin)
  • High C: small margin, few misclassifications — overfits
  • Low C: large margin, allows more errors — underfits
  • C is the regularization trade-off: margin size vs misclassification penalty
  • Tune C and γ via grid search with cross-validation
SVM vs Logistic Regression
  • SVM maximizes margin; LR maximizes log-likelihood
  • SVM more robust to outliers (soft margin focuses on support vectors)
  • LR gives calibrated probabilities; SVM doesn't natively
  • SVM harder to scale to large datasets (O(n²) to O(n³) training)
🎯 Interview Gold

"SVM with RBF kernel is powerful for small-to-medium datasets where the decision boundary is highly non-linear. Its key advantage: the dual formulation only depends on dot products, so the kernel trick allows working in infinite-dimensional spaces cheaply. Key disadvantage: doesn't scale — O(n²) memory and O(n³) training time. For n > 100K I'd use SGD-based linear SVM or switch to XGBoost. SVM also requires feature scaling and doesn't output well-calibrated probabilities."
