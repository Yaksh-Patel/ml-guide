Support Vector Machines (SVM)
Find the hyperplane that maximizes the margin between classes — geometrically principled, theoretically elegant, and surprisingly powerful in high dimensions.
Core Idea — Maximum Margin Hyperplane
$$\begin{aligned} &\text{Decision boundary: } \mathbf{w}^T\mathbf{x} + b = 0 \\ &\text{Margin} = \tfrac{2}{\|\mathbf{w}\|} \rightarrow \text{minimize } \|\mathbf{w}\|^2 \\ &\text{Hard margin: } \min \tfrac{1}{2}\|\mathbf{w}\|^2 \;\text{s.t.}\; y_i(\mathbf{w}^T\mathbf{x}_i+b) \geq 1 \\ &\text{Soft margin: } \min \tfrac{1}{2}\|\mathbf{w}\|^2 + C\textstyle\sum_i\xi_i \;\text{s.t.}\; y_i(\mathbf{w}^T\mathbf{x}_i+b) \geq 1-\xi_i \end{aligned}$$
Support vectors: the training points closest to the boundary — the only ones that matter. Remove any other point and the boundary doesn't change.
SVM: maximum margin hyperplane with support vectors
The Kernel Trick
Map features to higher-dimensional space where they become linearly separable — implicitly, without explicitly computing the mapping.
$$\begin{aligned} K(\mathbf{x},\mathbf{z}) &= \varphi(\mathbf{x})^T\varphi(\mathbf{z}) \quad \text{(inner product in feature space)} \\ \text{Linear:} &\quad K = \mathbf{x}^T\mathbf{z} \\ \text{Polynomial:} &\quad K = (\mathbf{x}^T\mathbf{z} + c)^d \\ \text{RBF/Gaussian:} &\quad K = \exp(-\gamma|\mathbf{x}-\mathbf{z}|^2) \quad\text{(most common)} \\ \text{Sigmoid:} &\quad K = \tanh(\alpha\mathbf{x}^T\mathbf{z} + c) \end{aligned}$$
RBF kernel: effectively infinite-dimensional feature space. γ controls radius of influence (high γ = narrow, overfits; low γ = wide, underfits).
C Parameter (Soft Margin)
  • High C: small margin, few misclassifications — overfits
  • Low C: large margin, allows more errors — underfits
  • C is the regularization trade-off: margin size vs misclassification penalty
  • Tune C and γ via grid search with cross-validation
Common Mistakes & Misconceptions
  • Not scaling features before SVM — SVM is the most scale-sensitive model: the margin calculation depends on Euclidean distances. Un-scaled features completely distort the decision boundary. Always StandardScaler
  • "C is regularization strength" — partially wrong. C is the inverse of regularization: large C = less regularization (harder margin, more likely to overfit); small C = more regularization (softer margin). This is opposite to sklearn's alpha in linear models
  • Using RBF kernel without tuning γ — γ controls the width of the RBF kernel. Too small: decision boundary too smooth (underfitting); too large: each training point becomes its own island (overfitting). Always tune C and γ together via grid search
  • Applying kernel SVM to large datasets — kernel SVM requires storing the kernel matrix (n×n) and solving a quadratic program: O(n²) memory and O(n³) time. For n > 50K, use LinearSVC (SGD-based) or switch to XGBoost
  • Expecting calibrated probabilities from SVM — SVM's decision function outputs a margin score, not a probability. sklearn's probability=True applies Platt scaling (fits a logistic regression on CV outputs) — expensive and imperfect. Use logistic regression if probabilities are needed
How It's Used at Top Companies
  • Computer vision (pre-deep learning era): SVM with histogram-of-oriented-gradients (HOG) features was the dominant object detection method until 2012; many production vision systems ran SVMs through the 2010s
  • Bioinformatics / genomics: still used for protein structure classification and cancer subtype prediction — datasets are small (hundreds of patients) but high-dimensional (thousands of gene expression features), exactly the SVM sweet spot
  • Text classification (small data): LinearSVC on TF-IDF features is a strong baseline for document classification; comparable to Naive Bayes but with a margin-based objective that often generalises better with more features
  • Anomaly detection (one-class SVM): One-Class SVM finds a decision boundary enclosing the "normal" class in kernel space; used for intrusion detection and manufacturing defect detection where anomalies are not present in training data
🎯 Interview Gold

What to say: "SVM with RBF kernel is powerful for small-to-medium datasets with non-linear boundaries — it implicitly works in a high-dimensional feature space via the kernel trick (avoid computing explicit feature maps). The margin maximisation objective gives it excellent generalisation on small datasets. Key constraints: O(n²–n³) training, O(n_sv × d) prediction (number of support vectors can be large). For n > 50K, LinearSVC scales linearly. Always StandardScale first — SVM is the most scale-sensitive common model."

Common mistake to avoid saying: "SVM outputs probabilities." — It doesn't natively. probability=True in sklearn adds costly Platt scaling. If you need calibrated probabilities, logistic regression is a better starting point.

Expect the follow-up: "What are support vectors?" — Answer: the subset of training points that lie on or within the margin boundary. Only these points determine the decision boundary — all other training points could be removed without changing it. The model's complexity is O(n_sv), not O(n), which is why SVMs generalise well: sparse representation of the decision boundary.
