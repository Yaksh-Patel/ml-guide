LLMs, SLMs & RLHF
The current frontier — massive language models that emergently acquire reasoning, coding, and instruction-following capabilities at scale. Understanding architecture, alignment, prompt engineering, hallucination, and agentic patterns is now the core applied ML skill.
Part 1 — Architecture & Scaling
Architecture of a Modern LLM
  • Decoder-only Transformer (GPT-style) — causal self-attention, each token attends to past only
  • RoPE positional encodings — rotates Q/K vectors by position; extends to longer contexts than learned embeddings
  • Grouped-Query Attention (GQA) — multiple Q heads share fewer K/V heads → faster inference, smaller KV cache
  • RMSNorm — drops mean subtraction vs LayerNorm; ~10% faster
  • SwiGLU activation in FFN — gated linear unit; empirically outperforms GELU
  • Flash Attention — IO-aware tiling: 4× faster attention, 10× less memory, no accuracy loss
Scaling Laws (Chinchilla)
$$\begin{aligned} \text{Chinchilla optimal: } N_{tokens} \approx 20 \times N_{params} \\ \text{Llama-3 8B trained on 15T tokens (5× Chinchilla)} \\ L \propto N^{-\alpha_N} + D^{-\alpha_D} + \text{irreducible noise} \end{aligned}$$
  • For a fixed compute budget: smaller model + more tokens beats larger model + fewer tokens
  • Emergent abilities: capabilities (CoT, in-context learning, arithmetic) appear abruptly at scale thresholds — not gradual
  • Modern SLMs deliberately overtrain past Chinchilla optimal for better inference-time efficiency (smaller model, same quality)
Part 2 — Alignment: RLHF, DPO & ORPO
RLHF — The Classic Pipeline
$$\begin{aligned} &\text{Stage 1: Pre-train on massive text (self-supervised next-token prediction)} \\ &\text{Stage 2: SFT on high-quality (prompt, response) demonstrations} \\ &\text{Stage 3: Train Reward Model on human preference pairs } (y_w \succ y_l) \\ &\text{Stage 4 PPO: } \max_{\pi_\theta}\,\mathbb{E}\!\left[r(y) - \beta\cdot\mathrm{KL}(\pi_\theta\|\pi_\text{SFT})\right] \end{aligned}$$
The KL penalty prevents "reward hacking" — without it the policy generates degenerate, high-scoring outputs that fool the reward model. PPO requires running the LLM during training (on-policy) — computationally expensive.
DPO — Direct Preference Optimization
$$\mathcal{L}_\text{DPO} = -\mathbb{E}\!\left[\log\sigma\!\left(\beta\log\frac{\pi_\theta(y_w|x)}{\pi_\text{ref}(y_w|x)} - \beta\log\frac{\pi_\theta(y_l|x)}{\pi_\text{ref}(y_l|x)}\right)\right]$$
  • Skips the separate reward model — solves for the optimal policy directly from preference pairs
  • y_w = preferred response; y_l = rejected response; β = deviation budget from π_ref
  • Simpler to train than PPO, often comparable quality
  • Weakness: offline — cannot explore new responses; requires high-quality paired preference data
ORPO — Odds Ratio Preference Optimization
$$\mathcal{L}_\text{ORPO} = \mathcal{L}_\text{SFT} - \lambda\cdot\mathbb{E}\!\left[\log\sigma\!\left(\log\frac{\text{odds}(y_w|x)}{\text{odds}(y_l|x)}\right)\right]$$
  • Merges SFT and preference alignment into a single training stage
  • No reference model needed — 50% less memory than DPO
  • Penalises rejected responses while rewarding preferred ones directly in the SFT loss
  • Widely used in Mistral, Phi-3, and resource-constrained fine-tuning pipelines

RAG Pipeline Design
Inject retrieved external knowledge into the context at inference time — reduces hallucination and keeps knowledge fresh without retraining.
  1. Chunk documents — fixed (512 tokens) or semantic boundaries
  2. Embed chunks — dense vectors (text-embedding-3, E5-large)
  3. Index in vector DB (FAISS, Pinecone, Weaviate, Chroma)
  4. At query time: embed query → top-k ANN retrieval → inject into prompt
  5. Optional rerank: cross-encoder scores query-chunk pairs → reorder top-k
  6. LLM generates with retrieved context as grounding

Part 4 — Hallucination: Causes & Mitigation
Why LLMs Hallucinate
  • Training objective mismatch: next-token prediction rewards fluency and plausibility, not factual accuracy
  • Knowledge gaps: sparse facts in training data get confabulated
  • Sycophancy: RLHF trains models to please humans — confidence even under uncertainty
  • Context blindness: model defaults to parametric memory over provided context
  • Temporal drift: world changes after training cutoff; model returns stale facts

Common Mistakes & Misconceptions
  • Confusing SFT and alignment — SFT (supervised fine-tuning) teaches format and instruction-following patterns from demonstration data. RLHF/DPO teaches values — which of two responses is preferred. You need both stages: SFT for capability, alignment for safety and quality
  • Fine-tuning to inject facts — fine-tuning encodes behavioural patterns, not reliable factual knowledge. Facts injected via SFT hallucinate when queried slightly differently. Use RAG for knowledge retrieval; fine-tune for reasoning style and output format
  • Treating temperature and top-p as equivalent — temperature scales the logit distribution (higher = flatter = more random). Top-p (nucleus sampling) truncates to the smallest set of tokens whose cumulative probability ≥ p, then samples from that set. Both add diversity, but top-p is more dynamic — it adapts to the sharpness of the distribution at each step. In practice, use temperature=0.7 + top-p=0.9 together; don't set both to extremes
  • Expecting deterministic outputs from LLMs — even at temperature=0, most implementations use greedy decoding which is deterministic, but small floating-point differences across hardware can cause divergence. For truly reproducible outputs, use seed + temperature=0 and note that exact determinism is hardware-dependent
  • Ignoring context window utilisation efficiency — LLMs attend to all tokens in context with O(n²) cost. Stuffing 100K tokens of marginally relevant context into the prompt increases latency quadratically. Retrieve only the most relevant chunks (RAG, top-k=3–5) rather than dumping entire documents
How It's Used at Top Companies
  • OpenAI (GPT-4 training pipeline): pretraining on web-scale data → SFT on high-quality demonstrations → reward model trained on human preference pairs → PPO fine-tuning maximising reward while penalising KL divergence from SFT model — this 3-stage RLHF pipeline is the blueprint for all frontier models
  • Anthropic (Constitutional AI): extends RLHF with AI feedback — a model critiques and revises its own outputs using a set of principles (the "constitution"); reduces dependence on human labellers for harmlessness and helpfulness signal
  • Meta (LLaMA 3 Instruct): DPO used instead of PPO for alignment — offline preference optimisation from preference pairs; simpler infrastructure (no reward model, no online rollouts), competitive performance with PPO at 8B–70B scale
  • Google (Gemini): RLHF with Gemini-based reward models; multimodal instruction following across text, images, video, audio — the reward model must evaluate preference across modalities, a significant extension of standard RLHF
🎯 Interview Gold

What to say: "The LLM training pipeline: pretraining (next-token prediction at scale) → SFT (teach instruction format from demonstrations) → alignment (RLHF or DPO, teach values from preference pairs). DPO is now preferred over PPO for most fine-tuning — no reward model needed, just preference pairs, single-stage offline optimisation. For production: 4-bit quantisation (AWQ/GPTQ) + vLLM for continuous batching + RAG for knowledge freshness. Hallucination is the core failure mode — mitigated by RAG grounding, structured outputs (JSON mode), and self-consistency sampling."

Common mistake to avoid saying: "I fine-tuned the model to update its knowledge." — Fine-tuning teaches behavioural patterns, not reliable factual knowledge. Injected facts hallucinate on paraphrase. Knowledge should live in a retrieval system (RAG), not in model weights.

Expect the follow-up: "When would you choose DPO over PPO?" — Answer: DPO for most fine-tuning use cases — no reward model to train and maintain, offline optimisation from existing preference pairs, single GPU feasible. PPO when you need to generate new rollouts online (the model can improve on trajectories it generates itself), when the reward signal is complex (multi-dimensional), or when you need maximum alignment performance at frontier scale where online RL signal drives the last 5–10% of quality.
