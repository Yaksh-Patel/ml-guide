Random Forest
Bootstrap aggregating applied to trees — reduces variance by averaging many diverse, decorrelated trees. The standard production workhorse for tabular data before XGBoost dethroned it.
Two Sources of Randomness (the key to diversity)
Bootstrap Sampling
Each tree trains on a bootstrap sample (~63.2% of data, with replacement). The ~36.8% left out = Out-of-Bag (OOB) data — a free validation set! OOB error ≈ k-fold CV error without the extra computation.
Feature Subsampling at each Split
At each node, consider only a random subset of √d features (classification) or d/3 (regression). Forces trees to be diverse — strong features don't always dominate. Decorrelated trees means their errors don't correlate → averaging reduces variance more.

Feature Importance — and its Biases
  • Impurity-based (MDI): mean impurity decrease across all trees. Fast but biased toward high-cardinality features and correlated features split importance between them.
  • Permutation importance: shuffle feature, measure OOB score drop. Model-agnostic, unbiased — but slower and correlated features still share importance.
  • SHAP: theoretically principled. Best overall — use TreeSHAP for speed.
📖 The Polling Analogy

Single tree = asking one opinionated, overconfident friend for a stock tip. Random Forest = polling 500 people with deliberately limited information (each saw different data and different features). Individual experts are imperfect but diverse — the collective vote is remarkably accurate. The diversity is the point.

🎯 Interview Gold

"Random Forest reduces variance via bagging — averaging over 500 independently-built trees. The feature subsampling decorrelates the trees, making the variance reduction much stronger than if trees were correlated. OOB error gives a free, reliable generalization estimate. For feature importance, I prefer permutation importance over impurity-based — it's unbiased toward high-cardinality features. For most tabular problems, start with RF then try XGBoost."
