Random Forest
Bootstrap aggregating applied to trees — reduces variance by averaging many diverse, decorrelated trees. The standard production workhorse for tabular data before XGBoost dethroned it.
Two Sources of Randomness (the key to diversity)
Bootstrap Sampling
Each tree trains on a bootstrap sample (~63.2% of data, with replacement). The ~36.8% left out = Out-of-Bag (OOB) data — a free validation set! OOB error ≈ k-fold CV error without the extra computation.
Feature Subsampling at each Split
At each node, consider only a random subset of √d features (classification) or d/3 (regression). Forces trees to be diverse — strong features don't always dominate. Decorrelated trees means their errors don't correlate → averaging reduces variance more.

Feature Importance — and its Biases
  • Impurity-based (MDI): mean impurity decrease across all trees. Fast but biased toward high-cardinality features and correlated features split importance between them.
  • Permutation importance: shuffle feature, measure OOB score drop. Model-agnostic, unbiased — but slower and correlated features still share importance.
  • SHAP: theoretically principled. Best overall — use TreeSHAP for speed.
📖 The Polling Analogy

Single tree = asking one opinionated, overconfident friend for a stock tip. Random Forest = polling 500 people with deliberately limited information (each saw different data and different features). Individual experts are imperfect but diverse — the collective vote is remarkably accurate. The diversity is the point.

Common Mistakes & Misconceptions
  • Tuning max_depth aggressively for Random Forest — unlike a single tree, Random Forest benefits from deep (even fully grown) trees because the ensemble variance reduction compensates. Default max_depth=None is often optimal; tune n_estimators and max_features first
  • Using impurity-based importance for correlated features — if two features are correlated, MDI importance splits arbitrarily between them. Use permutation importance or SHAP when features are correlated
  • Treating OOB error as identical to k-fold CV — OOB approximates 63.2%-trained models (bootstrap sees ~63.2% of data). It's a decent free estimate but slightly pessimistic vs true k-fold; for small datasets, run proper CV
  • Assuming more trees = overfitting — Random Forest does not overfit as n_estimators increases (unlike boosting). After a point, additional trees just cost time with negligible accuracy gain. 200–500 trees is usually sufficient
  • Using Random Forest for time series without careful CV — standard OOB / k-fold shuffles the timeline. Always use a time-based train/test split (no future leakage) for temporal data
How It's Used at Top Companies
  • Microsoft (Azure ML): Random Forest as the default baseline in AutoML pipelines — runs in parallel, robust to hyperparameter choices, and gives reliable feature importances for initial feature selection
  • Kaggle (pre-2014): Random Forest dominated structured-data competitions before XGBoost was released; still a reliable second model in ensembles
  • Healthcare (clinical risk models): used for patient readmission risk prediction where interpretability requirements are moderate and SHAP + permutation importance provide sufficient explanations for clinical staff
  • Bioinformatics: genomics feature selection via Random Forest importance — handle tens of thousands of genetic features (SNPs), where correlated feature groups are identified by clustering importance scores
  • Fraud detection: Random Forest as the reliable, low-maintenance baseline before deploying XGBoost/LightGBM — faster to train, easier to debug, and provides OOB estimate without separate validation set
🎯 Interview Gold

What to say: "Random Forest reduces variance via bagging — averaging over many independently-built trees. Two randomness sources are critical: bootstrap sampling (each tree sees ~63% of data) and feature subsampling at each split (√d features for classification). The feature subsampling decorrelates trees, making variance reduction much stronger than if trees were correlated. OOB error gives a free, reliable generalization estimate without a separate validation set. For feature importance, I default to permutation importance over impurity-based MDI — unbiased toward high-cardinality features."

Common mistake to avoid saying: "I set max_depth=5 in Random Forest to prevent overfitting." — RF doesn't overfit via depth the way single trees do. Reducing max_depth hurts accuracy without meaningful variance reduction benefit. Let trees grow deep.

Expect the follow-up: "When would you choose Random Forest over XGBoost?" — Answer: when you want a robust out-of-the-box model with minimal tuning, when OOB error is convenient, when training must run fully in parallel, or as a fast ensemble base. XGBoost usually wins accuracy contests but requires careful hyperparameter tuning and early stopping setup.
